# 슬라이드 17

제품 관리의 의미 : 제품이 고도화 됨에 따라 고객가치와 비즈니스 가치도 함께 상승할 수 있는 관리방식  문제정의 – 가설수립 – 배포 – 검증의 순환을 통한 점진적 고도화
제품관리 프로세스의 의미 : 제품의 목표설계부터 ‘제품 기획 – 배포- 유저임팩트 분석- 회고’ 의 순환과정을 아우르는 전반적인 프로세스
제품기획 산출물간 연계 : 목표관련 산출물 (결과지표  과정지표) 및 제품스펙 관련 산출물 (스토리  FLOW/고객여정맵  UX 와이어프레임  디자인) 간 연계
제품개발 협의 프로세스 : 우선순위 기준에 따라 관리되는 다양한 Task(백로그) 우선순위를 결정하는 프로세스
제품품질의 정의 : 제품의 목적의 만족도에 주어진 기간에 정해진 예산으로 제품이 생산되는 것
제품품질 관리방안 
  ㄴ 기획과정 참여하여 테스트케이스 작성  ㄴ 테스트 순환 횟수를 늘리는 방법  ㄴ 비즈니스과제와 기술부채/운영부채를 해결하는 과제 분리
제품배포 프로세스 : 개발 환경에서 짜인 코드가 최종적으로 운영 환경에서 실행되기까지 과정
릴리즈노트(배포노트) : 소프트웨어 또는 서비스가 출시 또는 업데이트 될 때마다 해당 상품의 배포와 함께 변경 사항, 기능 추가/삭제, 버그 개선 등 변경 사항을 체계적으로 정리하여 정보를 제공하는 문서
회고의 의미 : 지나간 일을 돌이켜 생각하는 것
KPT : 좋았던 부분을 유지(Keep)하고, 아쉬웠던 점이나 장애물을 제거(Problem)하여, 전체적인 퍼포먼스를 개선(Try)하는 회고 방법론
