## 1. 개념 정리 (경쟁사 Reference)

### 1.1 지점의 구조 변화

| 기존 구조 (2륜 지점) | 변경 구조 (NVP 권역/가상지점) |
|---|---|
| 상점 → 2륜 지점(기존지점) → 기사들 | 상점 → NVP 권역(가상지점) → 벤더들 → 기사들 |
| + 프렌즈 | + 프렌즈 |

### 1.2 세트 개념

**세트 정의:**
- 협력사에 물량을 위탁하는 최소 단위
- 요일/구간 별 물량으로 구성
  - 해당 물량 수행 성공여부와 거절률, 퀄리티 등을 정산 기준으로 삼음 (ex. 1주일 내 10구간 이상 성공)
- 1세트 수행을 위해 최소 액티브 기사를 N명으로 정의 중
- 협력사 마다 계약 세트 수를 탄력적으로 운영

→ **퀄리티 관리와 통제력 유지 수단으로 활용**

### 1.3 권역 운영

**리전 (클러스터의 집합)**
- 클러스터(픽업지) 단위로 상점-권역-협력사-기사 연결
- EDP(개인 배달 파트너)와 협력사 기사 혼재

### 1.4 정산 구조

| 구분 | 내용 |
|---|---|
| **쿠팡 → 협력사** | (1) 인센티브, (2) 기사 원가, (3) 기사 수행 실적 자료 |
| **협력사 → 기사** | 기사 원가의 자율적 배분 |

**인센티브:**
- 목표 지표 달성시 지급하기로 한 세트 당 협력사 fee

**기사 원가:**
- 쿠팡이 사전 설정한 지역별 배달 원가 + Dynamic Pricing

**구조 특징:**
- 기사앱상 표시되는 금액은 있으나 실제로 지급되지는 않으며 협력사에게 정산을 받는 방식
- 기사 관리에 힘을 실어줌 + 기사와 계약관계를 안가져 risk 감소

**지점장 수수료 (똥비/알비/관제비):**
- 지점장이 기사에게 오더당 일정 금액을 떼서 수취하기 위한 항목
- 기본 원가에서 설정한 N원을 차감 한 후 기사에게 표시

→ **지표에 따른 원가 지급을 통한 Controllability 확보 + 물량확보, 최적의 배차 제공 / 기사 수급 및 제안대로 성실히 수행 보장이라는 철저한 역할 분리**

### 1.5 정산 구조 상세

| 항목 | 내용 |
|---|---|
| **세트당 보상** | 목표 세트 지표 달성시 지급하기로 한 세트 당 협력사 fee. 협력사 지표에 따라 1세트도 3개 티어로 분화하여 관리 |
| **협력사 미션** | 협력사별 할당 세트를 넘어선 건수를 수행할 경우 지급하는 비용 |
| **협력사 등급 보상** | 협력사 별 티어 평가에 따른 주간 지급 비용 |
| **지역별 기사 원가** | 쿠팡에서 설정한 지역별 거리 원가 |
| **협력사 수수료** | 협력사장이 기사 원가에서 더하거나 차감하는 금액 |
| **협력사 미션** | 협력사가 소속 기사들에게 설정한 건수 미션 |
| **(실시간) 쿠팡 미션** | 퀄리티 관리를 위해 쿠팡에서 실시간 상황에 맞춰 추가한 미션 |

### 1.6 퀄리티 관리

**관리 지표 예시:**
- 세트 물량 수행
- 제안 오더 수락률
- (보조) 출근 기사 수
- (보조) 배송 퀄리티

**관리 방법:**

| 방법 | 내용 |
|---|---|
| **(1) 계약을 통한 관리** | 지급 물량 조정, 계약 해지 등 |
| **(2) 정산을 통한 관리** | 지표 도달 여부에 따른 인센티브 지급 |

→ **소단위 물량의 모듈화를 통한 쉬운 교체 + 주단위 실적에 따른 빠른 상벌 적용**

## 2. 벤더(협력사) 관리

### 2.1 원가 구조 (부릉-벤더 계약관계)

| 구분 | 기존 원가 구조 | 조건부 원가 구조 |
|---|---|---|
| **정의** | 수행완료한 관제 오더수에 비례하여 지점(벤더)장에게 금액을 지급하는 방식 | 일정 조건(배송 퀄리티, 수행 건수)의 만족 여부에 따라 지급 원가가 변동되는 방식 |
| **Pro** | 기존 지점 구조 전환 난이도 감소 | 원가를 통한 벤더 통제 레버 강화, 상호간 accountability 강화 (물량 영업, 적절한 배차 vs 배차된 오더의 수행, 기사 수급) |
| **Con** | 벤더/지점의 약한 accountability, Dynamic Pricing 기능에 대한 어뷰징 가능성 | 전환 방식에 대한 근본적 재논의 필요, 지역 관리자의 벤더 수급/관리 역량에 대한 의존성 |

### 2.2 권역의 관리 단위

| AS-IS | TO-BE |
|---|---|
| 중랑구 권역 전체를 하나로 관리 | 중랑 1, 2, 3, 4, 5 등 상점/오더 밀도에 맞춘 권역 세분화 |

**AS-IS 문제점:**
- 세밀한 수요 예측 / 공급 기획 어려움
- 벤더-프렌즈 물량 분배 비효율 발생 가능
- 예측/제안/세트 계약 등 이슈 발생 시 여파가 넓음

**TO-BE 장점:**
- 세밀한 수요 예측 / 공급 기획 가능
- 벤더/프렌즈 분포에 따른 물량 분배 정확도 상승
- 예측/제안/벤더 계약 이슈 발생 시 여파 contain 가능

### 2.3 추후 확정 필요 세부 정책들

| 내용 |
|---|
| 벤더와 권역의 매칭 기준 |
| 벤더의 수행오더 카운트 방식 |
| 정산 기준(관리지표) 설정 |
| 총판 개념 추가 필요 여부 |
| P0 개발 필요 사항 협의 |

## 3. 벤더 / 기사 등록과정

**프로세스:**
1. **세트 조직 계약** (세트 영업 담당자)
2. **벤더 조직 생성** + **벤더-지점 연결** (부릉 Intra, 권역 담당자)
3. **벤더 코드 발급** → 발급된 코드 전달
4. **벤더 장**: 앱 상 코드 입력 → 벤더장으로 등록 → 초대 코드 발급
5. **소속 기사**: 초대 코드 전달받음 → 앱 상 코드 입력 → 벤더 기사 등록 (시스템)

## 4. 벤더의 상태 관리

### 4.1 상태 정의

| 상태 | 설명 |
|---|---|
| **대기** | 사전에 영업하여 계약 대기중인 상태, 현재 투입중인 지점 대체 시 투입, 많이 확보할수록 통제력 강화 |
| **Active** | 벤더 기사로 시스템에서 인식하는 시점 (물량분배/원가 등 적용), 벤더는 1개 권역과만 이어질 수 있음 (대기상태에서는 연결 불가) |
| **해지** | 벤더 기사로 시스템에서 인식하는 시점 (물량분배/원가 등 적용), 벤더는 1개 권역과만 이어질 수 있음 (대기상태에서는 연결 불가) |

### 4.2 상태 변경 스케줄 (예시)

- 전주 조정 사항 적용 → 벤더 상태 조정 window (월~일) → 조정 사항 반영 (다음 주 월요일~)
- 조정 시 설정: 수행 세트 수 설정, 세트 조건 설정, 세트 보상 설정

## 5. 오더 분배

**구조:**
- 상점 → 오더 → NVP 지점(가상지점)
- 목표에 맞춘 분배 → 세트 계약 수 조정
- 세트조직(벤더들 + 기사들) + 프렌즈

→ **기계적 분배가 아닌 적정 후보 탐색 시 scoring 가중치를 주는 방식으로 분배 예정**

## 6. S&OP와 연계

### 6.1 주단위 Operation

**프로세스:**
1. 주간 물량 예측
2. 희망 Capa 비중 설정 (프렌즈 : Set 조직)
3. 원가 기획
4. 필요 Set 예측 → 투입 중 혹은 가능한 Set 조직 파악 → (필요시) 추가 Set 투입 또는 기존 Set 계약 해지
5. 필요 프렌즈 예측 → 현재 Active pool 파악 → (필요시) 프렌즈 프로모션 기획 또는 프렌즈 원가 인하/인상

**예시:**
- 주 10,000건 발생 예측 → 프렌즈 50% / 벤더 조직 50%
  - 6세트 필요 → 현황: 2세트 계약 벤더-1, 1세트 계약 벤더-1 → 3세트 수행을 위한 벤더 추가 계약
  - Active 프렌즈 20명 필요 → 현황: 지난주 액티브 프렌즈 15명 → 5명 추가 확보를 위한 프로모션 진행

### 6.2 실시간 Operation

**SLA/Capa 부족 예측 시:**
1. 실시간 배송 SLA 모니터링 (수요 대비 공급)
2. 실시간 Hourly 물량 예측
3. **1차 Capa 대응 (~15분)**: Dynamic Pricing (AI 배차), Set 수행 독려
4. **2차 Capa 대응 (~15분)**: 동적 미션 생성, Set 조직 추가 투입 / 본사 할증 적용 등

→ **실시간 혹은 준 실시간 모니터링 및 대응 체계 구축 필요**

## 7. 벤더 포탈

**구조:**
- 벤더 장 → 기사 계정 로그인 →
  - **현황 관리**: 세트 달성률, 기사 출/퇴근 현황, 기사별 수행률
  - **기사 관리**: 벤더 수수료 설정, 소속 기사 type 설정, 벤더 미션 설정
  - **+ 총판 계정**

---

## 8. 벤더 정산 기능

**프로세스:**
1. 설정된 조건 (수행 세트 수 설정, 세트 조건 설정, 세트 보상 설정, 기사 원가 설정)
2. 실적과 대조
3. 정산 금액 확정
4. 벤더 계좌로 지급

→ **주단위 정산 진행**
