## 3. PRD

### 3.1 벤더 등록기능

| 우선순위 | 기능 카테고리 | 유저 시나리오 | 개발 요구사항 | 레퍼런스 | Data 분석 조건 |
|:--------:|--------------|--------------|--------------|---------|---------------|
| P0 | 벤더 등록/관리 | 담당자는 벤더 정보(사업자 정보, 희망 지역, 계약서 등)를 등록할 수 있다 | **- 인트라 상 벤더 관리 메뉴를 추가한다.**<br>&nbsp;&nbsp;- 벤더 목록을 필터로 조회할 수 있다.<br>&nbsp;&nbsp;- 벤더를 신규 등록할 수 있다.<br>&nbsp;&nbsp;- 등록된 벤더를 삭제할 수 있다.<br><br>**- 벤더 등록 시 필수 입력 항목:**<br>&nbsp;&nbsp;- 사업자 정보 (상호명, 사업자등록번호, 대표자명)<br>&nbsp;&nbsp;- 희망 지역 (시군구 단위)<br>&nbsp;&nbsp;- 사업자등록증 이미지 (업로드)<br>&nbsp;&nbsp;- 계약서 이미지 (업로드)<br><br>**- 사업자등록번호 중복 검증:**<br>&nbsp;&nbsp;- 동일 사업자등록번호로 벤더 등록 시 Validation 오류 표시<br>&nbsp;&nbsp;- "이미 등록된 사업자입니다" 안내 | | |
| P0 | 벤더 등록/관리 | 담당자는 등록된 벤더 정보를 수정할 수 있다 | **- 벤더 상세 페이지에서 정보 수정이 가능하다.**<br>&nbsp;&nbsp;- 벤더 등록 정보 수정 가능<br>&nbsp;&nbsp;- 지점 연결 정보 수정 가능<br>&nbsp;&nbsp;- 물량 수주 정책 설정 가능 (논의 필요) | | |
| P0 | 벤더 상태 관리 | 담당자는 벤더의 상태를 대기/활성화/해지로 변경할 수 있다 | **- 벤더 상세 페이지에서 상태 관리 기능을 제공한다.**<br>&nbsp;&nbsp;- 상태 종류: 대기 / 활성화 / 해지<br>&nbsp;&nbsp;- 대기: 계약 대기 중인 상태, 권역 연결 불가<br>&nbsp;&nbsp;- 활성화: 시스템에서 인식하는 시점 (물량분배/원가 적용)<br>&nbsp;&nbsp;- 해지: 계약 종료 상태<br><br>**- 상태 변경 시 유의사항:**<br>&nbsp;&nbsp;- 대기 상태에서는 권역 연결이 불가하다.<br>&nbsp;&nbsp;- 활성화 상태의 벤더는 1개 권역과만 연결 가능하다.<br><br>**- 조직 이름 변경 기능:**<br>&nbsp;&nbsp;- 벤더 조직명을 변경할 수 있다. | | |
| P0 | 벤더-지점 연결 | 담당자는 벤더와 지점(권역)을 1:N으로 연결할 수 있다 | **- 벤더-지점 연결 기능:**<br>&nbsp;&nbsp;- 1개 지점에 여러 벤더를 연결할 수 있다.<br>&nbsp;&nbsp;- 벤더는 1개 지점(권역)과만 연결 가능하다. (1:N 관계)<br>&nbsp;&nbsp;- 연결된 벤더 목록을 조회할 수 있다.<br>&nbsp;&nbsp;- 연결을 해제(삭제)할 수 있다.<br><br>**- 연결 조건:**<br>&nbsp;&nbsp;- 벤더 상태가 '활성화'인 경우에만 연결 가능하다.<br>&nbsp;&nbsp;- 이미 다른 지점에 연결된 벤더는 연결 불가하다. | | |
| P0 | 벤더 코드 발급 | 담당자는 벤더장 등록용 코드를 발급할 수 있다 | **- 벤더 코드 발급 기능:**<br>&nbsp;&nbsp;- 벤더 상세 페이지에서 벤더 코드를 발급할 수 있다.<br>&nbsp;&nbsp;- 코드는 자동 생성된다.<br>&nbsp;&nbsp;- 발급된 코드를 플러스앱에서 입력하면 벤더장으로 자동 등록된다.<br><br>**- 코드 정책: (논의 필요)**<br>&nbsp;&nbsp;- 코드 유효기간<br>&nbsp;&nbsp;- 재발급 조건<br>&nbsp;&nbsp;- 코드 형식 (영문+숫자 조합 등) | | |
| P0 | 초대 코드 발급 | 담당자는 벤더 기사 등록용 초대 코드를 발급할 수 있다 | **- 초대 코드 발급 기능:**<br>&nbsp;&nbsp;- 벤더 상세 페이지에서 초대 코드를 발급할 수 있다.<br>&nbsp;&nbsp;- 추가 발급이 가능하다.<br>&nbsp;&nbsp;- 발급된 초대 코드를 플러스앱에서 입력하면 해당 벤더의 소속 기사로 등록된다.<br><br>**- 초대 코드 정책: (논의 필요)**<br>&nbsp;&nbsp;- 코드 유효기간<br>&nbsp;&nbsp;- 동시 사용 가능 횟수<br>&nbsp;&nbsp;- 재발급 조건 | | |

### 3.2 지점/권역관리

| 우선순위 | 기능 카테고리 | 유저 시나리오 | 개발 요구사항 | 레퍼런스 | Data 분석 조건 |
|:--------:|--------------|--------------|--------------|---------|---------------|
| P0 | 지점타입 추가 | 담당자는 지점 생성/수정 시 NVP-B 타입을 선택할 수 있다 | **- 지점 상세 페이지의 operation type에 NVP-B 타입 선택 옵션을 추가한다.**<br>&nbsp;&nbsp;- 기존 타입: A타입 지점<br>&nbsp;&nbsp;- 신규 타입: B타입 지점 (NVP-B)<br><br>**- NVP-B 타입 지점 저장 시 검증:**<br>&nbsp;&nbsp;- 타 지점의 sales 권역과 해당 지점의 세일즈 권역 중복 여부 체크<br>&nbsp;&nbsp;- 중복 시 저장 불가 처리<br>&nbsp;&nbsp;- "해당 권역은 이미 다른 지점에 등록되어 있습니다" 안내 | | |
| P0 | 권역 관리 설정 메뉴 신설 | 담당자는 NVP-B 지점별 설정값(세트 단위물량, 인센티브 조건 등)을 관리할 수 있다 | **- 권역 관리 설정 메뉴를 신설한다.**<br>&nbsp;&nbsp;- NVP-B 지점별 설정값 관리 목적<br><br>**- 관리 대상 설정값:**<br>&nbsp;&nbsp;- 세트 단위물량<br>&nbsp;&nbsp;- 세트 단위 인센티브 지급 조건<br>&nbsp;&nbsp;- 세트 단위 인센티브 금액<br><br>**- 설정값 관리 기능:**<br>&nbsp;&nbsp;- 설정값 생성<br>&nbsp;&nbsp;- 설정값 조회 (목록/상세)<br>&nbsp;&nbsp;- 설정값 수정<br>&nbsp;&nbsp;- 설정값 삭제 | | |
| P0 | 설점간-권역 연결 | 담당자는 설정값이 적용될 권역을 지정할 수 있다 | **- 연결 권역 탭을 제공한다.**<br>&nbsp;&nbsp;- NVP-B 타입 지점 검색 기능<br>&nbsp;&nbsp;- 다수의 권역 지정 기능<br><br>**- 권역 상세 설정:**<br>&nbsp;&nbsp;- 권역 상 설정값 선택 시 상세 페이지 진입 가능<br>&nbsp;&nbsp;- 권역 별 물량 및 수 입력/저장 기능<br><br>**- 연결 조건:**<br>&nbsp;&nbsp;- 동일 권역에 중복 설정값 적용 불가 (논의 필요) | | |

### 3.3 세트분배 관리

| 우선순위 | 기능 카테고리 | 유저 시나리오 | 개발 요구사항 | 레퍼런스 | Data 분석 조건 |
|:--------:|--------------|--------------|--------------|---------|---------------|
| P0 | 세트분배 관리 메뉴 신설 | 담당자는 NVP-B 지점별 세트 수 및 벤더별 세트 수를 확인할 수 있다 | **- 인트라 상 세트분배 관리 메뉴를 신설한다.**<br><br>**- 목록 화면:**<br>&nbsp;&nbsp;- NVP-B 지점 별 세트 수 표시<br>&nbsp;&nbsp;- 벤더별 분배된 세트 수 표시<br>&nbsp;&nbsp;- 지점 클릭 시 상세 페이지 진입<br><br>**- 상세 화면:**<br>&nbsp;&nbsp;- 해당 NVP 지점의 상세 정보 확인<br>&nbsp;&nbsp;- 연결된 벤더 목록 및 각 벤더별 세트 수 확인 | | |
| P0 | 권역 내 세트 물량 관리 | 담당자는 권역 내 목표 세트 수를 설정할 수 있다 | **- 권역 수준 분배 물량 설정 기능:**<br>&nbsp;&nbsp;- 월별 목표 세트 수 입력<br>&nbsp;&nbsp;- 벤더 상황에 따른 세트 수 조정<br><br>**- 세트 수 현황 표시:**<br>&nbsp;&nbsp;- 이미 분배된 세트 수 합계 표시<br>&nbsp;&nbsp;- 권역별 잔여 세트 수 표시<br>&nbsp;&nbsp;- 목표 대비 달성률 표시 (논의 필요) | | |
