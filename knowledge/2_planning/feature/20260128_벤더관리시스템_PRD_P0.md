## 3. PRD

### 3.1 벤더 등록기능

| 우선순위 | 기능 카테고리 | 유저 시나리오 | 개발 요구사항 |
|:--------:|-------------|-------------|-------------|
| P0 | 벤더 등록/관리 | 담당자는 벤더 정보(사업자 정보, 희망 지역, 계약서 등)를 등록할 수 있다 | **인트라 상 벤더 관리 메뉴 추가.** 벤더 목록 필터 조회 / 신규 등록 / 삭제 가능. **필수 입력 항목:** 사업자 정보(상호명, 사업자등록번호, 대표자명), 희망 지역(시군구 단위), 사업자등록증 이미지(업로드), 계약서 이미지(업로드). **사업자등록번호 중복 검증:** 동일 사업자등록번호 등록 시 Validation 오류 → "이미 등록된 사업자입니다" 안내 |
| P0 | 벤더 등록/관리 | 담당자는 등록된 벤더 정보를 수정할 수 있다 | **벤더 상세 페이지에서 정보 수정 가능.** 벤더 등록 정보 수정 / 지점 연결 정보 수정 / 물량 수주 정책 설정 가능 (논의 필요) |
| P0 | 벤더 상태 관리 | 담당자는 벤더의 상태를 대기/활성화/해지로 변경할 수 있다 | **벤더 상세 페이지에서 상태 관리 기능 제공.** 상태 종류: 대기(계약 대기, 권역 연결 불가) / 활성화(시스템 인식 시점, 물량분배·원가 적용) / 해지(계약 종료). **유의사항:** 대기 상태에서는 권역 연결 불가. 활성화 벤더는 1개 권역과만 연결 가능. **조직 이름 변경 기능:** 벤더 조직명 변경 가능 |
| P0 | 벤더-지점 연결 | 담당자는 벤더와 지점(권역)을 1:N으로 연결할 수 있다 | **벤더-지점 연결 기능:** 1개 지점에 여러 벤더 연결 가능. 벤더는 1개 지점(권역)과만 연결 가능(1:N 관계). 연결된 벤더 목록 조회 / 연결 해제(삭제) 가능. **연결 조건:** 벤더 상태 '활성화'인 경우에만 연결 가능. 이미 다른 지점에 연결된 벤더는 연결 불가 |
| P0 | 벤더 코드 발급 | 담당자는 벤더장 등록용 코드를 발급할 수 있다 | **벤더 코드 발급 기능:** 벤더 상세 페이지에서 코드 발급 가능. 코드는 자동 생성. 발급된 코드를 플러스앱에서 입력하면 벤더장으로 자동 등록. **코드 정책 (논의 필요):** 코드 유효기간 / 재발급 조건 / 코드 형식(영문+숫자 조합 등) |
| P0 | 초대 코드 발급 | 담당자는 벤더 기사 등록용 초대 코드를 발급할 수 있다 | **초대 코드 발급 기능:** 벤더 상세 페이지에서 초대 코드 발급 가능. 추가 발급 가능. 발급된 초대 코드를 플러스앱에서 입력하면 해당 벤더의 소속 기사로 등록. **초대 코드 정책 (논의 필요):** 코드 유효기간 / 동시 사용 가능 횟수 / 재발급 조건 |

### 3.2 지점/권역관리

| 우선순위 | 기능 카테고리 | 유저 시나리오 | 개발 요구사항 |
|:--------:|-------------|-------------|-------------|
| P0 | 지점타입 추가 | 담당자는 지점 생성/수정 시 NVP-B 타입을 선택할 수 있다 | **지점 상세 페이지의 operation type에 NVP-B 타입 선택 옵션 추가.** 기존 타입: A타입 지점 / 신규 타입: B타입 지점(NVP-B). **NVP-B 타입 저장 시 검증:** 타 지점의 sales 권역과 해당 지점의 세일즈 권역 중복 여부 체크. 중복 시 저장 불가 → "해당 권역은 이미 다른 지점에 등록되어 있습니다" 안내 |
| P0 | 권역 관리 설정 메뉴 신설 | 담당자는 NVP-B 지점별 설정값(세트 단위물량, 인센티브 조건 등)을 관리할 수 있다 | **권역 관리 설정 메뉴 신설.** NVP-B 지점별 설정값 관리 목적. **관리 대상 설정값:** 세트 단위물량 / 세트 단위 인센티브 지급 조건 / 세트 단위 인센티브 금액. **설정값 관리 기능:** 생성 / 조회(목록·상세) / 수정 / 삭제 |
| P0 | 설정값-권역 연결 | 담당자는 설정값이 적용될 권역을 지정할 수 있다 | **연결 권역 탭 제공.** NVP-B 타입 지점 검색 기능 / 다수의 권역 지정 기능. **권역 상세 설정:** 설정값 선택 시 상세 페이지 진입 가능. 권역별 물량 및 수 입력·저장 기능. **연결 조건:** 동일 권역에 중복 설정값 적용 불가 (논의 필요) |
| P0 | 세트 단위물량 관리 | 담당자는 권역별 세트의 단위물량을 설정할 수 있다 | **설정값 상세 화면에서 단위물량 관리.** 권역 관리 설정 상세에서 진입. 세트 보상 기준 = 최소 목표 달성 수행건 수취율(%) 입력 가능. **단위물량 설정:** 권역별 세트 단위의 물량 설정 기능. 설정값 상세 화면으로 진입하여 입력·저장 |
| P0 | 세트 보상 기준 관리 | 담당자는 권역 내 세트 물량 인센티브 지급 조건을 설정할 수 있다 | **각주 세트 보상 금액 정산 시 달성 조건 참조.** 설정값 상세 화면에서 진입. 최소 목표 달성 수행건 수취율(%) 입력 가능. **보상 기준 항목:** 세트 보상 기준별 최소 수행건 달성 조건 설정 / 제안 수락률(%) 입력 가능 |
| P0 | 세트 보상 금액 관리 | 담당자는 권역 내 세트별 인센티브 금액을 설정할 수 있다 | **세트 보상 기준 달성 시 지급할 인센티브 금액 입력.** 설정값 상세 화면에서 진입하여 세트 인센티브 금액 입력 가능. 0 초과 금액만 입력 가능. **삭제 시 처리:** 설정값이 삭제되면 세트 인센티브 지급도 중단 |
| P0 | 벤더 기사 요금제 관리 | 담당자는 이동거리(+action 비용) 기준 요금을 산정할 수 있다 | **요금제 기준 탭에서 기사 요금 산정.** 50m당 금액 / 픽업 비용 / 배송 센터 비용 각 건당 입력 가능. **요금 산정 기준:** 이동거리 + action 비용 기반 산정 |

### 3.3 세트분배 관리

| 우선순위 | 기능 카테고리 | 유저 시나리오 | 개발 요구사항 |

| P0 | 세트분배 관리 메뉴 신설 | 담당자는 NVP-B 지점별 세트 수 및 벤더별 세트 수를 확인할 수 있다 | **인트라 상 세트분배 관리 메뉴 신설.** **목록 화면:** NVP-B 지점별 세트 수 표시 / 벤더별 분배된 세트 수 표시 / 지점 클릭 시 상세 페이지 진입. **상세 화면:** 해당 NVP 지점의 상세 정보 확인 / 연결된 벤더 목록 및 각 벤더별 세트 수 확인 |

### 3.4 벤더 관리 기능 (플러스앱)

| 우선순위 | 기능 카테고리 | 유저 시나리오 | 개발 요구사항 |

| P0 | 로그인 제한 | 계약 세트 수에 따라 동시 로그인 가능한 기사 수가 제한된다 | **계약 세트 수 기반 동시 로그인 제한 기능:** 세트당 기사 수(초기 기준 5명) × 벤더에 부여된 세트 수 = 동시에 세트 기사로 활용 가능한 기사 수. 제한 초과 시 추가 기사는 세트 기사로 활용 불가. **소속 기사별 수행 모드 설정 기능:** 기사별 세트 기사 활동 여부 설정 가능 |

### 3.5 벤더 관리 기능 (벤더 포털 - 신규 페이지)

| 우선순위 | 기능 카테고리 | 유저 시나리오 | 개발 요구사항 |
|:--------:|-------------|-------------|-------------|
| P0 | 로그인 | 벤더장은 벤더 포털에 로그인하여 업무를 관리할 수 있다 | **벤더장 전용 업무관리 웹페이지 제공.** 벤더 담당자 등록 시 생성된 계정의 ID/PW를 활용하여 벤더 포털 로그인 가능. 로그인 후 현재 벤더의 소속 기사 현황 및 활동 상태 확인 가능 |
| P0 | 로그인 기사 관리 | 벤더장은 소속 기사의 세트 기사 활동 여부를 관리할 수 있다 | **소속 기사 활동 관리 기능:** 현재 세트 기사 활동 가능 수 이외의 기사가 트래킹을 열 경우, 해당 기사의 세트 기사 활동 가능 여부를 설정 가능. 활동 가능/불가능 기사 목록 조회 기능 [확인 필요] |
| P0 | 대시보드 (리포팅) | 벤더장은 벤더별 미션/세트 배송 수행 현황을 실시간으로 조회할 수 있다 | **자동 정산 및 결과 리뷰 기능 (리포팅):** 조회일 기준 벤더별 미션 세트의 총 배송 수행 건수를 실시간으로 조회 가능. 목표 달성 여부 확인 가능 [확인 필요] |

### 3.7 벤더 포털 - 세트/물량 관리 (신규 페이지)

| P0 | 확정 세트 단위물량 신청 | 벤더장은 권역별 세트 단위물량을 신청할 수 있다 | **확정 세트 단위물량 신청 기능:** 권역별 세트 단위물량 작성 가능 / NVP 지점 권역과 권역 물량에 대한 요청 기능 / 이전 history 기반 적정 세트 수를 참고하여 신청 가능. **참고 데이터:** 과거 오더 history 기반 적정 세트 운용량 참조 |

### 3.8 S&OP 대시보드

| 우선순위 | 기능 카테고리 | 유저 시나리오 | 개발 요구사항 |
