# 벤더관리 시스템 — 용어집

> 원본: [01 용어집 — 벤더관리 시스템 (노션)](https://www.notion.so/325cf5759dc081a181fccecaa3d5bea3)

## 액터 정의

| 한국어 | 코드 | 정의 | 사용 채널 | 주요 행동 |
|--------|------|------|----------|----------|
| 리전 매니저 | RM | 본사에서 벤더, 세트, 지점 등을 총괄 관리하는 담당자 | 인트라넷 | 벤더 등록/해지, VO 지정, 지점 연결, 정산 컨펌 |
| 벤더장 | VO (Vendor Operator) | 벤더를 대표하여 소속 기사와 실적을 관리하는 현장 책임자 | 벤더 포털 | 기사 승인/해제, 실적 확인, 단위물량 변경 신청, 정산 조회 |
| 세트 기사 | DP-S (Delivery Partner - Set) | 벤더 소속으로 세트 배송에 참여하는 기사. 출근 시 세트 자리가 남아있으면 자동 배정 | 플러스앱 | 세트 물량 배차 수락/수행, 벤더장 통해 정산 |
| 프렌즈 기사 (벤더 소속) | DP-F (Delivery Partner - Friends) | 벤더에 소속되어 있지만 세트에 참여하지 않는 기사. 세트 자리 초과 시 자동 전환 | 플러스앱 | 프렌즈 배차 수행, M캐시 즉시 정산 |
| 프렌즈 기사 (미소속) | DP-FA (Delivery Partner - Friends Alone) | 벤더에 소속되지 않은 일반 기사. 장애 시 모든 기사가 이 모드로 전환 | 플러스앱 | 프렌즈 배차 수행, M캐시 즉시 정산 |

## 도메인 용어

### 벤더/조직

| 한국어 | 영어 | 정의 | 관련 기능 |
|--------|------|------|----------|
| 벤더 | Vendor | 배송을 맡아서 하는 외부 업체. 본사와 계약을 맺고 B존 지역에서 배송 서비스를 수행 | 벤더 관리 |
| 벤더장 | VO (Vendor Operator) | 벤더의 현장 책임자. 기사 관리와 실적 확인 담당. 벤더당 1명만 지정 가능 | 벤더 관리, 벤더 포털 |
| EDP | External Delivery Partner | 외부 배송 기사. 벤더 소속 기사를 지칭하는 통칭 | 플러스앱 |
| 협력사 | Partner | 벤더의 다른 표현. 배송 물량을 위탁받아 수행하는 외부 업체 | 전체 |

### 벤더 상태

| 한국어 | 코드 | 정의 | 할 수 있는 것 |
|--------|------|------|-------------|
| 대기 | STANDBY | 벤더가 등록 완료 후 운영 준비 중인 상태 | VO 지정, 기사 등록 준비 |
| 운영중 | ACTIVE | 벤더가 정상 운영 중인 상태 | 지점 연결, 세트 배분, 배송 참여, VO 지정, 기사 등록 모두 가능 |
| 운영중지 | INACTIVE | 벤더가 일시적으로 운영을 중단한 상태 | 세트 할당 회수, 기사 소속 유지 |
| 해지 | TERMINATED | 계약이 종료된 상태. 소속 기사 전원 자동 탈퇴 | 이력 보존. 동일 사업자번호로 재등록 가능 |

### 기사 소속 상태

| 한국어 | 코드 | 정의 |
|--------|------|------|
| 승인 대기 | PENDING | 기사가 초대 코드로 가입 신청 후, VO의 승인을 기다리는 상태 |
| 활성 | ACTIVE | VO가 승인 완료. 벤더 소속으로 배송 참여 가능 |
| 거절 | DECLINED | VO가 가입 요청을 거절한 상태 |
| 비활성 | INACTIVE | VO가 기사를 벤더에서 내보낸 상태. DP-FA로 전환 |

### 세트/물량

| 한국어 | 영어 | 정의 | 관련 기능 |
|--------|------|------|----------|
| 세트 | Set | 협력사에 물량을 위탁하는 최소 단위. 요일/구간별 물량으로 구성. 퀄리티 관리와 통제력 유지 수단 | 대시보드 |
| 슬롯 | Set Period (Set Slot) | 하루를 5개 시간대로 나눈 구간. 물량 배분과 실적 측정의 기본 단위 | 대시보드 |
| 단위물량 매트릭스 | Unit Volume Matrix | 7일(요일) × 5슬롯으로 구성된 35칸 물량 표. 세트 1개의 기본 물량 구성 | 단위물량 변경 |
| 세트당 기사 수 | Riders per Set | 1세트를 수행하기 위해 필요한 최대 동시 활성 기사 수. 권역관리 관리정책의 세트 물량 구성 탭에서 설정 | 기사 관리 |

**슬롯 구분:**

| 아침 | 07:00~11:00 |
| 점심 피크 | 11:00~14:00 |
| 점심 피크 후 | 14:00~17:00 |
| 저녁 피크 | 17:00~20:00 |
| 저녁 피크 후 | 20:00~23:00 |

### 지역/권역

| 한국어 | 영어 | 정의 | 관련 기능 |
|--------|------|------|----------|
| B존 | B-Zone | 라이더 인프라가 취약하여 벤더를 통해 배송을 운영하는 지역 | 전체 |
| 클러스터 | Cluster | 픽업지 기준으로 묶은 지역 단위. 물량 예측의 기본 단위 | 지점/권역관리 |
| NVP 권역 | NVP Zone | 벤더가 운영하는 가상지점 기반 권역 | 지점/권역관리 |
| 벤더존 | VendorZone (NVP-B) | B존 벤더 운영을 위한 지점 타입. 기존 지점 관리에 추가되는 신규 타입 | 지점/권역관리 |
| 존 | Zone | 배송 운영이 계획·관리되는 범위 | 전체 |

### 배차/배송

| 한국어 | 영어 | 정의 | 관련 기능 |
|--------|------|------|----------|
| 배차 | Dispatch | 주문이 발생했을 때 적합한 기사에게 배송을 할당하는 과정 | 플러스앱 |
| 배차 오퍼 | Dispatch Offer | 기사에게 전달되는 배송 제안. 수락/거절/만료(30초) 가능 | 플러스앱 |
| dispatchSource | Dispatch Source | 배차 시점에 결정되는 배차 유형. SET(세트 배차) 또는 FRIENDS(프렌즈 배차) | 플러스앱 |
| 에스컬레이션 | Escalation | DP-S 풀 후보 소진 시 Friends 풀로 배차 대상을 자동 확대하는 것 | 플러스앱 |
| Vendorset | Vendorset | 벤더 세트 배차를 관리하는 시스템 모듈. DP-S 후보 기사 목록을 제공 | 플러스앱 |
| Dispatch Service | Dispatch Service | 통합 배차 엔진. Vendorset에서 후보를 받아 가중치 기반으로 기사를 선정 | 플러스앱 |

### 정산/비용

| 한국어 | 영어 | 정의 | 관련 기능 |
|--------|------|------|----------|
| 벤더 수수료 | Vendor Fee | 벤더장이 기사 원가에서 더하거나 차감하는 금액. 벤더장의 수익원 | 정산 조회 |
| 기사 원가 | Rider Cost | 본사에서 설정한 지역별 거리 기반 배송 원가. 기사에게 표시되는 배달료의 기준 | 정산 조회 |
| M캐시 | M-Cash | 기사 앱의 수익금 잔액. DP-F/DP-FA는 배송 완료 즉시 입금, DP-S는 미입금(벤더 정산) | 플러스앱 |
| V캐시 | V-Cash | 벤더 기사 전용 가상 포인트. 정산 예정 금액을 보여주며 출금 불가 | 플러스앱 |
| 세트당 보상 | Set Reward | 목표 세트 지표 달성 시 본사가 벤더에게 지급하는 세트당 fee | 정산 조회 |
| 관리수수료 | Management Fee | 계약조건(달성률/수행률 등) 충족 시 SET당 지급하는 인센티브 금액 | 정산 조회 |
| 멀티배달 환산 | Multi-delivery Conversion | 멀티배달 1건을 SET상 0.8건으로 환산하는 기준 | 정산 조회 |

### 시스템/운영

| 한국어 | 영어 | 정의 | 관련 기능 |
|--------|------|------|----------|
| 인트라넷 | Intranet | 본사 내부 관리 시스템. RM이 벤더/지점/세트를 관리하는 웹 어플리케이션 | 벤더 관리, 지점관리 |
| 벤더 포털 | Vendor Portal | 벤더장(VO) 전용 웹 어플리케이션. 기사 관리, 실적 조회, 정산 확인 등 | 벤더 포털 |
| 플러스앱 | Plus App | 배송 기사용 모바일 앱. 벤더 가입, 출근, 배차 수락, 배송 수행 등 | 플러스앱 |
| 초대 코드 | Invitation Code | 기사가 벤더에 가입할 때 입력하는 코드. 벤더당 1개 자동 생성, 만료 없음 | 벤더 관리, 플러스앱 |
| 모드 자동 결정 | Auto Mode Decision | 기사 출근 시 우선순위에 따라 DP-S/DP-F/DP-FA를 자동 판정하는 로직 | 플러스앱 |
| 폴백 | Fallback | Vendorset 장애 시 모든 기사를 DP-FA로 전환하여 프렌즈 배차로 배송을 지속하는 비상 대응 | 플러스앱 |

### 실적/지표

| 한국어 | 영어 | 정의 | 관련 기능 |
|--------|------|------|----------|
| 달성률 | Achievement Rate | 슬롯별 목표 물량 대비 실제 수행 완료 비율 | 대시보드 |
| 수락률 | Acceptance Rate | 배차 제안 대비 기사가 수락한 비율 | 대시보드 |

## 약어 정리

| 약어 | 풀네임 | 의미 |
|------|--------|------|
| RM | Region Manager | 리전 매니저 (본사 담당자) |
| VO | Vendor Operator | 벤더장 (현장 책임자) |
| DP-S | Delivery Partner - Set | 세트 기사 |
| DP-F | Delivery Partner - Friends | 프렌즈 기사 (벤더 소속) |
| DP-FA | Delivery Partner - Friends Alone | 프렌즈 기사 (미소속) |
| EDP | External Delivery Partner | 외부 배송 기사 |
| NVP | Network Vendor Partner | 벤더 파트너 네트워크 |
| NVP-B | NVP B-type | B존 벤더 운영 지점 타입 |
| VCC | VROONG Customer Center | 부릉 고객센터 |
| SLA | Service Level Agreement | 서비스 수준 합의 |
| DLQ | Dead Letter Queue | 실패 이벤트 임시 보관 큐 |
