## 개요

라이더 인프라가 취약한 B존 지역에 본사가 직접 세트지점/벤더 구조를 구축하여 운영하는 전략(NVP-B)을 실행하기 위해, 벤더 등록부터 벤더 포털, 기사 등록까지 전 과정을 시스템화합니다.

- **벤더 운영 체계**: 벤더 등록/상태 관리/지점 연결/VO 지정을 인트라넷에서 표준화
- **벤더 포털**: 벤더장(VO)이 소속 기사/실적을 직접 관리하는 신규 웹
- **기사 등록/모드 결정**: 플러스앱에서 벤더 기사 등록 + 출근 시 자동 모드 결정

## AS-IS vs TO-BE

| 항목 | 현재 상황 | 문제점 | TO-BE 변경 내용 |
|------|----------|--------|----------------|
| 벤더 등록/관리 | 수작업 (엑셀/구두) | 표준화 안 됨, 상태 추적 불가 | **인트라넷에서 벤더 CRUD + 상태 관리 + 지점 연결** |
| VO/기사 소속 관리 | 수작업 | 프로세스 미표준화 | **RM이 VO 지정, 초대 코드 기반 기사 등록, VO가 포털에서 승인** |
| 벤더장 업무 관리 | 없음 | 실적/정산 확인 채널 없음 | **벤더 포털 (기사 관리, 대시보드, 정산 조회)** |
| 기사 모드 결정 | 수작업 | 세트/프렌즈 구분 불가 | **출근 시 자동 모드 결정 (DP-S/DP-F/DP-FA)** |
