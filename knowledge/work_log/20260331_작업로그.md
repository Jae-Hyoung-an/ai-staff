**온라인 목업:** https://jae-hyoung-an.github.io/ai-staff/vendor_zone_mockup.html

**완료된 작업 이력 (시간순):**

1. **버전 관리 + 예약 변경 기능 추가**
   - 권역관리: 변경 이력 → 버전 관리 탭으로 전환, 예약 변경 탭 추가
   - 세트분배관리: 동일 구조 적용
   - 버전 상세/비교는 팝업 모달, 예약은 날짜+시간 지정 가능

2. **벤더 상태 관리 추가**
   - 벤더 상태 종류: 대기 / 운영중 / 운영중지 / 해지
   - 4개 탭 전체에 벤더 상태 배지 표시
   - 비운영 벤더 세트 제한 (0만 허용), 예약 적용 시 상태 보정

3. **기사 원가 요금제 이관** (가장 최근)
   - 권역관리: 기사 원가 요금제 탭 삭제 (4탭 → 3탭)
   - 세트분배관리: '적용 기사원가' read-only 탭 추가 (요금제 서버 연동)
   - 벤더기사물량분배: 원가 참조 경로를 관리정책 → 요금제 서버로 변경

4. **세트당 최대 활성화 기사수 추가**
   - 권역관리 세트 물량 구성 탭 하단에 입력 필드 추가 (최소 1, 최대 99)
   - 벤더별 최대 활성화 기사수 = 벤더 분배 세트 수 × 이 값
   - 벤더 포털(벤더관리 시스템 PRD 2.6)에서 세트 모드 기사 상한으로 사용

## 주요 참조 관계

```
권역관리 관리정책
  ├── 세트 물량 구성 (7x5 매트릭스 + 세트당 최대 활성화 기사수)
  ├── 보상기준
  └── 연결 zone
        └── 세트분배관리 (Zone별)
              ├── 연결 벤더 + 벤더별 세트 분배
              ├── 적용 기사원가 (요금제 서버 read-only)
              └── 벤더기사물량분배 (배차 엔진)
                    └── 원가 참조: 요금제 서버

벤더관리 시스템 (별도 PRD)
  ├── 벤더 관리 (인트라) — 벤더 등록/상태/VO 지정
  ├── 벤더 포털 — 기사 관리 (세트 가능 인원 = 세트당 최대 활성화 기사수 × 할당 세트 수)
  └── 플러스앱 — 출근 시 모드 자동 결정
```
