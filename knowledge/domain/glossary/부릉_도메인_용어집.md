# 부릉 도메인 용어집

> 부릉(Vroong) 서비스 운영에서 사용되는 핵심 용어 정리
> 출처: NVP 정책서, 프로젝트 산출물, 기능명세서 등
> 최종 업데이트: 2026-04-02

---

## 플랫폼 및 앱

| 용어 | 정의 | 비고 |
|------|------|------|
| NVP | New Vroong Platform. 직고용(DIRECTLY_EMPLOYED) 기사 전용 배달 플랫폼 | 부릉플러스앱의 기반 |
| 부릉플러스앱 | NVP 기사 전용 배달 수행 앱. 클라이언트 식별자 `riderplus` (AGENT_NEW_APP) | |
| 기사M | 이륜 기사 전용 앱. 클라이언트 식별자 `agentapp` (AGENT_APP) | FULL_TIME, PART_TIME |
| 프렌즈 앱 | Vroong Friends 전용 앱. 클라이언트 식별자 `vroongfriends` (FRIENDS_APP) | 유연근무 배달 파트너 |
| 인트라 | 내부 관리자 시스템. 상점/기사/오더/정산/할증 등 관리 | 운영팀·관리자가 사용 |
| 매니저PC/M | 지점 매니저 전용 관리 도구 | 기상할증 ON/OFF 등 |

## 조직 및 운영

| 용어 | 정의 | 비고 |

| 테스트베드 | 신규 앱/기능 런칭 전 실환경 테스트를 위한 지정 지점 | 예: 중랑직영센터 |

## 오더 및 배차

| 용어 | 정의 | 비고 |
|------|------|------|
| 오더 | 고객의 배달 주문 1건 | |
| NVP 오더 | 오더 생성 상점의 관제 지점이 NVP 지점인 오더 | |
| 일반 오더 | 기본 배송 유형. 여러 오더 동시 수행 가능 | |
| 단건 오더 | 한 번에 하나의 주문만 수행. 동시수행 불가 | |
| 묶음 오더 | 여러 건을 하나로 묶어 기사에게 동시 제안. 순서대로 수행 | |
| 과적 오더 | 무게/양이 많아 추가 운임이 필요한 경우 원오더에 추가로 생성되는 오더 | |
| 원오더 | 과적 오더의 기준이 되는 원래 오더 | |
| 동시제안 오더 | AI 배차가 하나의 오더를 여러 기사에게 동시에 제안하는 오더 | |
| 화주사 연동 오더 | 외부 화주사와 시스템 연동되어 생성 (올리브영, 스타벅스 등) | |
| 묶음배차 | 여러 오더를 하나의 묶음으로 만들어 기사에게 한번에 제안하는 배차 방식 | |
| AI 추천배차 | AI 배차엔진이 기사에게 최적의 오더를 자동으로 제안하는 시스템 | |
| 강제 배차 | 관리자가 기사에게 오더를 직접 배정하는 것 | |
| 배차 취소 | 기사가 배정된 오더를 취소하는 행위. 일 3회 제한 | |

## 오더 수행 흐름

| 용어 | 정의 | 비고 |
|------|------|------|
| 오더 제안 On/Off | 기사가 AI 추천배차 후보 등록 여부를 선택하는 토글 | Default: off |
| 오더 수락 | 기사가 제안된 오더를 수락 (30초 타이머 내) | |
| 상점 도착 | 기사가 상점에 도착했음을 앱에서 처리. 반경 100m 검증 | |
| 픽업 완료 | 상점에서 물건 수령 후 앱에서 완료 처리 | |
| 배달 완료 | 고객에게 물건 전달 후 앱에서 완료 처리 | |
| 재이동 | 픽업 중 or 배송 완료 후 도착지 변경 시 신규 수행지로 재배송 | |
| 반납 | 배송 불가 판단 시 물건을 상점으로 돌려보내는 처리 | |
| 수행 Task | 기사가 처리해야 할 개별 작업 단위 (픽업, 배달, 반납 등) | |

## 권역 및 분배

| 용어 | 정의 | 비고 |
|------|------|------|
| 권역 (Zone) | 오더 분배의 지리적 단위. 지점 하위에 복수 권역 설정 가능 | |
| 관리정책 | 권역에 적용되는 오더 분배 규칙 (요일/시간대별 설정) | |

| 벤더기사물량분배 | 벤더 소속 기사에게 오더를 분배할 때의 물량/원가/우선도 로직 | AI 배차 연동 |
