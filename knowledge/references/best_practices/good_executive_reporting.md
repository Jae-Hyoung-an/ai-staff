# [Best Practice] 좋은 경영진 보고

> **핵심 메시지**: 경영진 보고의 목적은 정보 전달이 아니라 **의사결정 유도**다.  
> 5분 안에 핵심을 파악하고, 결정을 내릴 수 있게 구조화해야 한다.

---

## 핵심 요약

경영진은 수많은 정보를 빠르게 판단해야 한다. 보고서가 길고 복잡하면 핵심이 묻힌다. **"결론 먼저, 근거는 나중에"**, **"요청 사항 명확히"**, **"숫자로 말하기"**가 경영진 보고의 3대 원칙이다. 좋은 보고는 듣는 사람이 아닌 **읽는 사람 관점**에서 설계된다.

---

## 1. 경영진 보고의 원칙

### 1.1 핵심 원칙

```
1. BLUF (Bottom Line Up Front)
   → 결론/핵심 메시지를 가장 먼저

2. So What?
   → 모든 정보에 "그래서 뭐?"에 대한 답 포함

3. One Page
   → 핵심은 한 페이지 안에 (상세는 Appendix)

4. Action Required
   → 무엇을 결정/승인해 달라는 것인지 명확히
```

### 1.2 경영진이 원하는 것

| 경영진 니즈 | 보고서에 반영 |
|------------|--------------|
| **빠른 판단** | 핵심 요약, 결론 먼저 |
| **리스크 파악** | 문제점과 대응 방안 |
| **의사결정** | 명확한 요청 사항 |
| **신뢰** | 데이터 기반 근거 |
| **전략 정렬** | 비즈니스 목표와 연결 |

## 2. 보고서 구조

### 2.1 기본 구조 (One Page Summary)

```
┌────────────────────────────────────────────────────────────┐
│                    [보고서 제목]                            │
│              2024년 Q4 결제 전환율 개선 결과                 │
├────────────────────────────────────────────────────────────┤
│                                                            │
│  📌 Executive Summary                                      │
│  ─────────────────────────────────────────────            │
│  • 결론: 간편결제 도입으로 전환율 7%p 상승 (목표 5%p 달성)   │
│  • 매출 영향: 월 2.1억원 추가 매출 발생                      │
│  • 요청: 추가 결제 수단 확대 예산 승인 (5천만원)             │
│                                                            │
│  📊 핵심 지표                                               │
│  ─────────────────────────────────────────────            │
│  • 결제 전환율: 58% → 65% (+7%p)                           │
│  • 결제 소요시간: 45초 → 28초 (-38%)                        │
│  • 이탈률: 32% → 22% (-10%p)                               │
│                                                            │
│  ⚠️ 리스크 & 이슈                                          │
│  ─────────────────────────────────────────────            │
│  • 토스 연동 비용 예상보다 30% 초과 (대응: 네고 완료)        │
│  • iOS 결제 오류 3건 발생 (대응: 핫픽스 배포 완료)           │
│                                                            │
│  📅 Next Steps                                             │
│  ─────────────────────────────────────────────            │
│  • 1월: 네이버페이 연동 착수                                │
│  • 2월: 페이코, 삼성페이 연동                               │
│                                                            │
│  ───────────────────────────────────────────────          │
│  * 상세 데이터: Appendix 참조                               │
└────────────────────────────────────────────────────────────┘
```

### 2.2 보고 유형별 구조

**진행 상황 보고 (Status Update)**

```
1. 핵심 상태 (On Track / At Risk / Delayed)
2. 주요 성과 (지난 기간)
3. 다음 계획 (향후 기간)
4. 이슈 & 대응
5. 요청 사항 (있는 경우)
```

**의사결정 요청 (Decision Request)**

```
1. 결정이 필요한 사항 (명확하게)
2. 배경 (왜 이 결정이 필요한지)
3. 옵션 비교 (장단점, 비용, 리스크)
4. 추천안 및 근거
5. 결정 데드라인
```

**문제 보고 (Problem Report)**

```
1. 문제 요약 (무엇이, 얼마나 심각한지)
2. 영향 범위 (비즈니스 임팩트)
3. 원인 분석
4. 대응 현황 (이미 한 것)
5. 추가 대응 계획 (할 것)
6. 요청 사항 (필요한 지원)
```

## 3. 효과적인 커뮤니케이션

### 3.1 숫자로 말하기

**Before (모호)**
```
"전환율이 많이 개선되었습니다.
사용자 만족도도 높아졌습니다."
```

**After (명확)**
```
"전환율이 58%에서 65%로 7%p 상승했습니다.
이는 목표 대비 140% 달성이며,
월 2.1억원의 추가 매출에 해당합니다."
```

### 3.2 비교 기준 제시

```
[나쁜 예]
"DAU가 10만명입니다."
→ 많은 건지 적은 건지 모름

[좋은 예]
"DAU가 10만명으로 전월 대비 15% 상승했습니다.
업계 평균(8만)보다 25% 높은 수준입니다."
```

### 3.3 시각화 활용

| 데이터 유형 | 적합한 차트 | 부적합한 차트 |
|------------|-----------|--------------|
| **시간별 변화** | 라인 차트 | 파이 차트 |
| **비교** | 바 차트 | 에어리어 차트 |
| **구성 비율** | 파이/도넛 | 라인 차트 |
| **분포** | 히스토그램 | 파이 차트 |
| **두 변수 관계** | 산점도 | 바 차트 |

## 4. 스토리텔링 기법

### 4.1 Situation-Complication-Resolution (SCR)

```
[Situation] 상황
"현재 결제 전환율은 58%로 업계 평균(62%)보다 낮습니다."

[Complication] 문제/도전
"경쟁사들이 간편결제를 도입하면서 격차가 벌어지고 있습니다.
이대로면 Q4 매출 목표 달성이 어렵습니다."

[Resolution] 해결책
"간편결제 도입으로 전환율 65%를 목표합니다.
예상 추가 매출은 월 2억원입니다.
개발 비용 5천만원, ROI는 3개월입니다."
```

### 4.2 Pyramid Principle (Minto)

```
                    [결론]
              전환율 개선 필요
                    │
        ┌───────────┼───────────┐
        ▼           ▼           ▼
    [근거1]      [근거2]      [근거3]
   전환율 낮음   경쟁 심화    매출 영향
        │           │           │
    ┌───┴───┐   ┌───┴───┐   ┌───┴───┐
    ▼       ▼   ▼       ▼   ▼       ▼
  데이터   비교  경쟁사   시장  수치   예측
         기준   분석   트렌드  분석

→ Top-down: 결론부터, 근거는 계층적으로
```

## 6. 발표 스킬

### 6.1 발표 구조

```
[도입 - 1분]
"오늘 3가지를 공유드립니다:
1) Q4 성과
2) 발견한 기회
3) 승인 요청 드릴 사항"

[본론 - 8분]
• 핵심 메시지 3개 이내
• 각 메시지에 근거 데이터
• 시각 자료 활용

[결론 - 1분]
"정리하면...
오늘 결정 요청 드리는 것은...
질문 있으시면..."
```

### 6.2 질문 대응

**예상 질문 준비**

```
[예상 질문 목록]
Q1: "왜 이 방법이 최선인가?"
→ 대안 비교 자료 준비

Q2: "리스크는?"
→ 리스크 목록과 대응 방안 준비

Q3: "경쟁사는 어떻게 하고 있나?"
→ 벤치마킹 자료 준비

Q4: "숫자의 근거는?"
→ 데이터 소스와 계산 방식 준비
```

**모르는 질문에 대응**

```
[좋은 응답]
"정확한 데이터를 확인한 후 오늘 중으로 공유드리겠습니다."

[피해야 할 응답]
"글쎄요... 아마... 그럴 것 같습니다."
```

## 7. 피해야 할 안티패턴

### ❌ 안티패턴 1: 과정 나열

```
나쁜 예:
"1월에 기획을 했고, 2월에 디자인을 했고,
3월에 개발을 시작해서 4월에 QA를 완료했습니다."

좋은 예:
"4개월 만에 출시했고, 전환율이 7%p 올랐습니다.
핵심 성공 요인은 간편결제 도입이었습니다."
```

### ❌ 안티패턴 2: 결론 없는 데이터 나열

```
나쁜 예:
"DAU 10만, MAU 50만, 전환율 12%, 
NPS 45, 리텐션 30%입니다."
→ 그래서?

좋은 예:
"전환율 12%가 목표 대비 낮습니다.
결제 단계 이탈이 주원인이며,
간편결제 도입으로 15%까지 개선 가능합니다."
```

### ❌ 안티패턴 3: 변명으로 시작

```
나쁜 예:
"리소스가 부족해서, 일정이 촉박해서,
요구사항이 바뀌어서... 결과가 좋지 않습니다."

좋은 예:
"목표 대비 80% 달성했습니다.
미달 원인은 A이고, 대응 방안은 B입니다.
추가 지원이 필요한 사항은 C입니다."
```

### ❌ 안티패턴 4: 한 슬라이드에 모든 것

```
문제:
• 글자가 작아서 안 보임
• 핵심이 무엇인지 불명확
• 집중 분산

해결:
• 한 슬라이드 = 한 메시지
• 상세 내용은 Appendix로
• "이 슬라이드에서 기억할 것은..."
```

## 8. 체크리스트

### 보고서 작성 전
- [ ] 이 보고의 목적(정보 공유/의사결정/승인)이 명확한가?
- [ ] 청중(경영진)이 알아야 할 핵심은 무엇인가?
- [ ] 요청 사항이 있다면 무엇인가?

### 보고서 작성 중
- [ ] 결론/핵심이 가장 먼저 나오는가?
- [ ] 모든 주장에 데이터 근거가 있는가?
- [ ] "그래서 뭐?"에 답하고 있는가?
- [ ] 한 페이지 안에 핵심이 담기는가?

### 발표 전
- [ ] 예상 질문과 답변을 준비했는가?
- [ ] 시간 내 발표 가능한가? (리허설)
- [ ] Backup 자료(Appendix)가 준비되어 있는가?

## 2. 주요 성과
- **배달 시간 개선**: AI 배차 v2 출시 후 피크 시간대 34분→31분
- **취소율 감소**: 예상 시간 정확도 개선 효과

## 3. 주요 이슈

- **대응**: 
  - 단기: 피크 시간 인센티브 10% 인상 (승인 필요)
  - 중기: 라이더 등급제 도입 검토

## 4. 다음 주 계획
| 항목 | 목표 | 담당 |

- [ ] 피크 인센티브 10% 인상 승인 (예산: 월 2천만원, 기한: 1/23)

### 예시 2: 분기 성과 보고 (배달 플랫폼)

```markdown
# 배달 플랫폼 Q4 2025 성과 보고

## Executive Summary
- **핵심 성과**: 전환율 7%p 개선, 분기 목표 매출 107% 달성
- **주요 이슈**: 라이더 공급 부족으로 배달 시간 목표 미달
- **요청 사항**: Q1 라이더 확보 마케팅 예산 3억원 승인

---

## 1. 비즈니스 KPI 달성 현황

| KPI | 목표 | 실적 | 달성률 |
|-----|------|------|--------|
| GMV | 500억 | 535억 | **107%** |
| 전환율 | 62% | 65% | **105%** |
| 평균 배달 시간 | 28분 | 31분 | **90%** |
| 월 활성 주문자 | 150만 | 158만 | **105%** |
| 라이더 유지율 | 85% | 82% | **96%** |

---

## 2. 핵심 성과

### 성과 1: 간편결제 도입 → 전환율 7%p 상승
- 투자: 5천만원 (개발 + 연동비)
- 효과: 월 2.1억원 추가 매출
- ROI: 2개월

### 성과 2: 검색 알고리즘 개선 → 검색 전환율 15% 상승
- 개인화 추천 적용 비율: 45% → 78%
- 검색 후 주문 전환율: 12% → 13.8%

---

## 3. 미달성 항목 분석

### 평균 배달 시간 (목표: 28분, 실적: 31분)
**원인 분석**:
1. 라이더 공급 부족 (목표 대비 -12%)
2. 동시 주문 급증 시 배차 지연
3. 날씨/교통 상황 반영 미흡

**대응 계획**:
- Q1 라이더 확보 캠페인 (목표: +2,000명)
- AI 배차 고도화 (동시 주문 처리 최적화)

---

## 4. Q1 2026 중점 계획

| 우선순위 | 이니셔티브 | 목표 | 투자 |
|---------|-----------|------|------|
| P1 | 라이더 확보 캠페인 | +2,000명 | 3억원 |
| P1 | 결제 수단 확대 | 전환율 +2%p | 5천만원 |
| P2 | 배달 시간 예측 고도화 | 정확도 90% | 2억원 |

---

## 5. 요청 사항

### 의사결정 필요
| 항목 | 내용 | 금액 | 기한 |
|------|------|------|------|
| 라이더 확보 예산 | 마케팅 + 인센티브 | 3억원 | 1/31 |
| 추가 결제 수단 | 네이버페이/페이코 | 5천만원 | 1/31 |

### 권고안
라이더 확보 예산 승인 권고
- 근거: 현재 라이더 부족이 배달 시간의 주 원인
- 기대 효과: 배달 시간 28분 달성, NPS 5점 상승
```
