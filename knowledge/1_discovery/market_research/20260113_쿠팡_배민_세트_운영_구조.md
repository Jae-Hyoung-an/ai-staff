---

## 쿠팡 (쿠팡이츠 포함)

| 구분 | 내용 |
|------|------|
| **물류 인프라** | 전국 40개+ 물류센터 (200만㎡+), 도심 외곽 배치 |
| **배송 인력** | 쿠팡친구 (자체 배송 인력), 직접 고용 형태 |
| **배차 방식** | AI 알고리즘 기반 수요 예측 및 배차 최적화 |
| **운영 특징** | 직매입 → 자체 보관 → 자체 배송 (End-to-End 통제) |

### 추정되는 권역 운영

- 물류센터(허브) 중심의 스포크 구조
- 배송 권역별 쿠팡친구 배치
- 실시간 배차 최적화

---

## 배민 (배달의민족)

| 구분 | 내용 |
|------|------|
| **플랫폼 역할** | 음식점 ↔ 소비자 연결 (중개 플랫폼) |
| **배송 인력** | 배민라이더스 (자체) + 배민커넥트 (긱워커) |
| **배차 방식** | AI 배차 시스템, 라이더 선택형 병행 |
| **운영 특징** | 플랫폼 중심, 가맹점 배달 인프라 활용 |

### 추정되는 권역 운영

- 지역/구 단위 권역 설정
- 권역별 라이더 풀 운영
- 주문 밀집도에 따른 동적 배차

## 일반적인 배달 플랫폼의 "세트" 운영 구조

```
┌─────────────────────────────────────────┐
│              본사/플랫폼                  │
│         (배차 알고리즘, 정책 관리)         │
└─────────────────┬───────────────────────┘
                  │
     ┌────────────┼────────────┐
     ▼            ▼            ▼
┌─────────┐ ┌─────────┐ ┌─────────┐
│ 세트 A  │ │ 세트 B  │ │ 세트 C  │
│ (강남)  │ │ (송파)  │ │ (서초)  │
└────┬────┘ └────┬────┘ └────┬────┘
     │           │           │
     ▼           ▼           ▼
 ┌───────┐   ┌───────┐   ┌───────┐
 │라이더 │   │라이더 │   │라이더 │
 │그룹   │   │그룹   │   │그룹   │
 └───────┘   └───────┘   └───────┘
```

| 구성 요소 | 설명 |
|----------|------|
| **세트 (권역)** | 지리적으로 묶인 배송 구역 단위 |
| **라이더 그룹** | 해당 권역에 배치된 라이더 풀 |
| **거점/캠프** | 라이더 대기 및 관리 장소 (있는 경우) |
| **배차 로직** | 권역 내 주문 → 해당 라이더 그룹에 배차 |
