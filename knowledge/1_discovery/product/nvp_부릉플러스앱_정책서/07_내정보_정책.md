## 1. 내 정보 개요

기사가 본인의 등록 정보, 인증 현황, 계정 설정을 열람·관리.

### 1-1. 진입 조건

| 항목 | 내용 |
|------|------|
| 진입 경로 | 마이페이지 → 내 정보 |
| 접근 대상 | 로그인된 기사 |
| 화면 구성 | 5개 섹션: 기사 정보 / 운행 정보 / 개인 정보 / 기타 정보 / 계정 정보 + 로그아웃 기능 |

### 1-2. 화면 섹션 구성

| 섹션 | 표시 항목 | 비고 |
|------|-----------|------|
| 기사 정보 | 활동명, 기사코드, 배달수단 | 읽기 전용 |
| 운행 정보 | 면허 정보, 안전모, 범죄경력회보서, 고용/산재보험 인증 현황 | 각 항목별 인증 상태 표시 |
| 개인 정보 | 본인인증 현황, 실명, 전화번호, 주소 | 본인인증 미완료 시 인증 필요 표시 |
| 기타 정보 | 시간제보험 가입 현황 | 툴팁 및 이동 버튼 포함 |
| 계정 정보 | 계정 ID, 비밀번호 변경 | 이동 버튼 포함 |

## 2. 운행 정보

기사의 4가지 인증 항목 현황 표시, 미완료 항목 인증 유도.

### 2-1. 인증 항목

면허 정보 / 안전모 / 범죄경력회보서 / 고용·산재보험

### 2-2. 인증 상태 정의

| 상태 | 표시 텍스트 | 설명 |
|------|-------------|------|
| 미인증 | 인증 필요 | 인증 미진행 |
| 보류 | 검증 중 | 등록 후 승인/반려 전 |
| 반려 | 재인증 필요 | 검토 후 반려 |
| 승인 | 인증 완료 | 정상 인증 완료 |
| 미해당 | 미해당 | 안전모 - 비필수 기사 OR 면허 비필수 / 고용·산재보험 - 계약구분 ≠ 일반 OR 활동형태 ∉ (풀타임, 파트타임) OR 지점타입 ≠ LASTMILE |

### 2-3. 인증 유도 배너 표시 조건

4개 항목 중 1개 이상 `인증 완료`·`검증 중(보류)`가 아닌 항목 존재 시 → 인증 유도 배너 표시 (구 기사앱 **인증하기** 버튼)

### 2-4. 보류 배너 표시 조건

면허정보, 안전모, 범죄경력회보서 중 1개 이상 보류 중 → 보류 배너 표시

### 2-5. 배너 노출 우선순위

| 우선순위 | 배너 |
|----------|------|
| 1순위 | 인증 유도 배너 |
| 2순위 | 검증중 배너 |

## 3. 개인 정보

기사 본인의 신원·연락처 정보 표시, 본인인증 미완료 시 유도.

### 3-1. 표시 항목

본인인증 현황 / 실명(읽기 전용) / 전화번호(읽기 전용) / 주소(읽기 전용)

### 3-2. 본인인증 미완료 시

| 조건 | 처리 |
|------|------|
| 본인인증 미진행 | `개인 정보 인증이 필요해요` 배너 표시, **인증하기** → 기사M 이동 |
| 본인인증 완료 | 배너 미표시 |

## 4. 기타 정보

시간제보험 가입 현황 표시, 미가입 시 가입 유도.

| 상태 | 표시 조건 | 표시 정책 |
|------|-----------|-----------|
| 미가입 | 시간제 보험 미가입 | `미가입` |

| 가입 + 적용 | 가입 AND M캐시 ≥ 보증금 | `적용` (시간당 단가, 일최대한도) |
