# [NVP] 컴플라이언스 정책

> 원본: Notion - [NVP] 컴플라이언스 정책서 (강수은, 리뷰중)
> 최종 업데이트: 2026-03-24

---

## 1. 컴플라이언스 개요

NVP 기사가 오더를 수행하기 위해 사전에 충족해야 하는 자격 요건 목록과 검증 체계.

### 1-1. 오더 수행 사전 절차 목록

| # | 항목 | 필수 여부 | 검증 시점 |
|---|------|-----------|-----------|
| 1 | 본인인증 | 필수 | 로그인 완료 시 |
| 2 | 필수 약관 동의 | 필수 | 로그인 완료 시 |
| 3 | 고용/산재보험 | 필수 | 로그인 완료 시 |
| 4 | 면허 정보 | 필수 ('필수' 기사) | 로그인 완료 시 |
| 5 | 안전모 | 필수 ('필수' 기사) | 로그인 완료 시 |
| 6 | 범죄경력회보서 | 필수 | 로그인 완료 시 |

### 1-2. 인증 플로우

```
NVP 통합기사앱 로그인 → 미인증 정보 안내 → 구 기사앱 이동
  → 구 기사앱 설치 확인 (미설치 시 플레이스토어 이동)
  → 구 기사앱 로그인 → 본인 인증 진행 → 부릉 서비스 이용 약관 5개 동의
```

## 2. 필수 약관 동의

부릉 플러스 서비스 이용을 위해 기사가 약관에 동의하는 절차.

| 항목 | 정책 |
|------|------|
| 동의 대상 | 서비스 이용약관 / 위치정보 수집 및 이용 / 개인정보 수집 및 이용 / 고객 개인정보 보호 서약 / M캐시 조정기능 동의서 |
| 동의 판단 기준 | `termsAgreedAt` 값이 존재하면 동의 완료로 간주 |
| 약관 변경 시 재동의 | **[미확인]** 약관 버전 관리 방식/재동의 로직 미확인 |

---

## 3. 본인인증

운전면허 인증의 전제 조건으로 작동하는 기사 본인 확인 절차.

| 항목 | 정책 |
|------|------|
| 인증 방식 | NICE 실명확인 서비스를 통한 이름 + 생년월일 확인 |
| 인증 필수 시점 | 운전면허 인증 전 |
| 본인인증 검증 시점 | 운전면허 인증 요청 시 (미완료이면 면허 인증 불가) |
| 저장 정보 | 이름, 생년월일, 고유 개인식별값(CI) |

---

## 4. 운전면허 인증

기사의 운전면허 등록, 인증 상태 관리, 유효성 판단 기준.

### 4-1. 인증 상태

| 상태 | 설명 | 오더 배정 가능 |
|------|------|----------------|
| 승인 | 면허 인증 완료 | 가능 |
| 보류 | 심사 중 | 가능 |
| 반려 | 면허 인증 거절 | **불가** |
| 미등록 | 면허 등록 자체가 없음 | 면허 필요 기사인 경우 불가 |

### 4-2. 등록 정보

- 이름, 면허증 번호 (암호화 저장), 면허 종류, 운전면허 고유식별정보 처리 동의

### 4-3. 안내사항

1. 복수 면허 시 한 개만 선택
2. 125cc 초과 이륜차 운행 시 2종 소형 면허 필요
3. 외국인 국제면허 등록은 VCC(1800-8255) 문의
4. 타인 면허 등록 시 불이익은 본인 책임

## 5. 범죄경력회보서

기사의 범죄경력회보서 제출 대상, 기한, 상태 관리.

| 항목 | 정책 |
|------|------|
| 제출 대상 | NVP 기사 전원 |
| 제출 기한 | 첫 주문 수행일로부터 **14일** |
| 반려 후 재제출 기한 | 반려일로부터 **14일** |
| 부적합 처리 | 생활물류법 제19조의2에 해당하는 범죄경력 조회 시 오더 수행 불가 |

## 6. 고용보험 / 산재보험

### 6-1. 일반기사 고용보험 대상 조건

| 조건 | 값 |
|------|-----|
| 파트너 타입 | lastmile |
| 계약 유형 | general |
| 고용 형태 | 풀타임 또는 파트타임 |

### 6-2. 보험 제외 대상

| 제외 조건 | 제외 대상 보험 |
|-----------|----------------|
| 65세 이상 | 고용보험만 제외 |
| 내국인, F2, F5, F6이 아닌 비자 | 고용보험 + 산재보험 모두 제외 |

### 6-3. 전일분 일일 정산 정책

1. **거래 발생**: D-1일 하루 동안 기사들의 M캐시 거래 발생
2. **전일 거래 추출**: D일 새벽 - 전일 거래를 4구간으로 분할 추출 후 정산 시작
3. **보험료 계산**: 대상 기사 조회 → 기사별/지점별 청구서 생성 → 지점 변경 기사 보정
4. **M캐시 이체**: 기사 및 지점 대상 M캐시 이체 실행

## 7. 필수 정보 상태 변경 처리

인트라 또는 매니저PC에서 컴플라이언스 관련 정보가 변경될 때 앱 내 처리 방식.

### 7-1. 변경 주체별 대상

| 변경 주체 | 상태변경 대상 | 초기화 대상 |
|-----------|---------------|-------------|
| 인트라 | 면허 상태변경, 안전모 상태변경 | 고용/산재보험 초기화, 본인인증 정보 초기화 |
| 매니저PC | 범죄경력회보서 상태변경 | - |

### 7-2. 변경 발생 시 앱 처리

| 앱 상태 | 처리 방식 |
|---------|-----------|
| 오더 대기 중 (오더 받기 on) | 오더 받기 off 전환 + 재인증 요청 팝업 |
| 오더 수행 중 | 오더 받기 off 전환 + 배차된 오더는 수행 가능 → 완료 후 재인증 요청 팝업 |
