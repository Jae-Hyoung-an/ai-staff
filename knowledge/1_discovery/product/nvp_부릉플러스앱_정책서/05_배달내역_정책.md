# [NVP] 배달내역 정책

> 원본: Notion - [NVP] 배달내역 정책서 (강수은, 리뷰중)
> 최종 업데이트: 2026-03-24

---

## 1. 배달내역

기사가 기간을 선택하여 완료한 배달의 건수·수익 합계와 오더 목록을 조회.

### 1-1. 진입 조건

| 항목 | 내용 |
|------|------|
| 진입 경로 | 마이페이지 > 배달 내역 |
| 접근 가능 대상 | 로그인된 기사 |
| Default 화면 | '오늘' 배달 내역 탭 |

### 1-2. 기간 탭 정의

| 탭 | 조회 범위 |
|-----|-----------|
| 오늘 | 조회 시점의 당일 (00:00~23:59) |
| 주별 보기 | 조회 시점이 속한 주의 일요일~토요일 |

### 1-3. 배달 건수, 수입 정보

| 항목 | 집계 기준 |
|------|-----------|
| 완료 배달건 | 선택 기간 내 완료·취소된 건수 합산 |
| 수입 | 시스템 상 즉시 확정·계산 금액 합산 (기본 배달료, 취소 수수료, 미션 보상금, 기타 즉시 정산 가능 요금) |

### 1-4. Empty 케이스

| 탭 | 정책 |
|-----|------|
| 오늘 | '배달하러 가기' 버튼 노출 → tap 시 메인화면 이동 |
| 주별 보기 | empty 화면 노출 |

---

## 2. 배달 상세

개별 오더 선택 시 상세 정보, 수수료·결제 내역 제공.

### 2-1. 표시 항목

| 항목 | 정책 |
|------|------|
| 상점명 | 인트라에 등록된 상점명 / OD 오더: 화주사 전달 '출발지' 정보 |
| 오더 종류/상태 태그 | 상태와 상관없이 태그 복수 노출 가능 |
| 수행 완료 일시 | yyyy.mm.dd hh:mm |
| 픽-도 직선 거리 | - |
| 부릉 오더번호 | - |
| 배달지 | 고객 정보 만료 시 미노출 |
| 배송료 | 취소된 오더도 배송료 표시 |
| 기상할증 | 기상할증 붙은 오더에 한해 노출 |
| 합계 | 배송료 + (기상할증) |
| 결제내역 버튼 | 대상: 후불 오더 / 노출 기준: 배달 완료·취소 후 24시간 이내 |

---

## 3. 후불 결제 처리

후불 오더에서 발생한 잔여 결제 금액을 기사가 직접 결제하는 흐름.

### 3-1. 잔여 결제 금액 노출 조건

| 조건 | 표시 |
|------|------|
| 후불 오더에 미수금 남아있는 경우 | 경고문구 |
| 후불 오더에 미수금 + 배달 완료/취소 후 24시간 이내 | 경고문구 + 결제하기 버튼 |

### 3-2. 결제내역 버튼 노출 조건

| 항목 | 정책 |
|------|------|
| 대상 오더 | 후불 오더 |
| 노출 기준 | 배달 완료/취소 시간 기준 24시간 이내 |

### 3-3. 후불 현금 결제 내역

| 케이스 | 표시 내용 | 기능 |
|--------|-----------|------|
| default | 결제일시, 결제금액, `현금` 라벨 | 결제 취소 요청 (VCC 연결) / 현금영수증 발행 (KIS Pay) |
| 결제 취소 | 결제일시, 취소일시, 취소선 + `현금 취소` 라벨 | 결제 취소 불가 / 현금영수증 불가 |
| 취소 후 재결제 | 취소 내역(상단) + 신규 결제 내역(하단) 동시 표시 | 결제 취소 요청 (VCC) / 현금영수증 발행 (KIS Pay) |

### 3-4. 후불 카드 결제 내역

| 케이스 | 표시 내용 | 기능 |
|--------|-----------|------|
| default | 결제일시, 카드종류, 카드번호(마스킹), 승인번호, `카드` 라벨 | 영수증 확인 / 결제 취소 |
| 결제 취소 | 결제일시, 취소일시, 카드종류, 카드번호(마스킹), 승인번호, 취소선 + `카드 취소` 라벨 | 영수증 확인만 가능 |
| 취소 후 재결제 | 취소 내역(상단) + 신규 카드 결제 내역(하단) 동시 표시 | 취소건: 영수증 확인 / 재결제건: 영수증 확인 + 결제 취소 |
