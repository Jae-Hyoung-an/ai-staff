## 1. 로그인 가능 조건

NVP 통합기사앱에 로그인할 수 있는 기사의 유형, 상태, 소속 지점 조건.

### 1-1. 앱 타입별 허용 기사 유형

| 앱 | 클라이언트 식별자 | 고용형태 (EmploymentType) | 비고 |
|-----|-------------------|---------------------------|------|
| NVP 통합기사앱 | `riderplus` (AGENT_NEW_APP) | `DIRECTLY_EMPLOYED` | NVP 전용 |
| 기사M | `agentapp` (AGENT_APP) | `FULL_TIME`, `PART_TIME` | 이륜 기사 전용 |
| 프렌즈 앱 | `vroongfriends` (FRIENDS_APP) | `VROONG_FRIENDS` | 프렌즈 전용 |

### 1-2. NVP 앱 기사 상태별 로그인 제한

| 기사 상태 | 로그인 가능 | 안내 메시지 |
|-----------|-------------|-------------|
| 활성화 | O | - |
| 퇴직 | X | "퇴직 상태로 로그인 할 수 없습니다" |
| 휴직 | X | "휴직 상태로 로그인 할 수 없습니다. 관리자에게 문의해 주세요." / 자동 휴직 시 VCC(1800-8255) 안내 |
| 삭제 | X | "이미 삭제된 계정으로 로그인할 수 없습니다." |

### 1-3. NVP ↔ 기사M 타입 로그인 제한

- NVP 앱 로그인은 **지점 Operation Type = NVP** 기사인 경우에만 가능
- 비NVP 기사: `"해당 앱을 이용할 수 없는 기사타입 입니다."` 에러 메시지 노출

## 2. 비밀번호 정책

### 2-1. 비밀번호 생성 규칙

| No. | 규칙 | 상세 |
|-----|------|------|
| 1 | 길이 제한 | 최소 10자, 최대 20자 |
| 2 | 문자 조합 | 영문과 숫자 혼합 필수 |
| 3 | 허용 문자 | 영문, 숫자, 특수문자만 사용 가능 |
| 4 | 연속 문자 제한 | 동일/연속 숫자·문자 4자 이상 연속 불가 |
| 5 | 이전 비밀번호 재사용 | 현재 비밀번호와 동일한 비밀번호 설정 불가 |

- 연속 문자 검증: 증가 연속(abcd, 1234) / 감소 연속(dcba, 4321) 4자 이상 불가
- 허용 특수문자: `` \ ~ ! @ # $ % ^ & * ( ) - = _ + [ ] { } : ; ' , . / < > ? \ | ``

### 2-2. 비밀번호 만료 정책

| 항목 | 정책 |
|------|------|
| 만료 기간 | **90일** |
| 만료 판단 기준 | 마지막 비밀번호 변경일 + 90일 |
| 만료 시 처리 | 로그인 시 비밀번호 변경 화면으로 강제 이동 |

### 2-3. 비밀번호 변경

| 항목 | 정책 |
|------|------|
| 변경 가능 경로 | 내 정보 > 계정 정보 > 비밀번호 변경 / 만료 시 강제 노출 |
| 현재 비밀번호 확인 | 필수 (현재 비밀번호 입력 후 검증) |
| 새 비밀번호 규칙 | 2-1. 생성 규칙과 동일 |
| 변경 후 처리 | **모든 기기에서 강제 로그아웃** |
| 과거 비밀번호 재사용 | 직전 1개(현재 비밀번호)만 불가 |

## 3. IP 접근 제한

기사 계정별로 허용된 IP에서만 로그인할 수 있도록 접근을 제한.

| 항목 | 정책 | 비고 |
|------|------|------|
| IP 등록 방식 | 기사 계정별 허용 IP 등록 | 관리자가 설정 |
| 등록된 IP가 없는 경우 | 모든 IP에서 로그인 허용 | - |
| 등록된 IP가 있는 경우 | 해당 IP / CIDR 대역만 허용 | - |
| CIDR 지원 | 지원함 | 예: '10.0.0.0/24' |
| 검증 시점 | 로그인 요청 시 | - |
| 미충족 시 | "로그인이 불가능한 IP입니다." | - |

## 4. 세션/토큰 관리

로그인 이후 세션 유지, 토큰 갱신, 자동 로그인 해제 정책.

### 4-1. 토큰 유효 기간

| 토큰 종류 | 유효 기간 |
|-----------|-----------|
| Access Token | **1시간** |
| Refresh Token | **90일** |

### 4-2. 자동 로그인

- 유지 조건: Refresh Token 유효 기간(90일) 내 자동 갱신

### 4-3. 자동 로그인이 해제되는 경우

| 항목 | 설명 |
|------|------|
| 수동 로그아웃 | 기사 직접 로그아웃 / 관리자가 인트라에서 강제 실행 |
| 비밀번호 변경 | 변경 즉시 세션 무효화 |
| 비밀번호 만료 | 만료 기간(90일) 초과 시 |
| Refresh Token 만료 | 90일 경과 시 |
| 지점 타입 변경 | NVP ↔ 기존 지점 전환 시 |
| 사용자 정보 변경 | **[미확인]** 어떤 사용자 정보 변경 시 해제되는지 확인 필요 |
| 재직 상태 변경 | **[미확인]** 어떤 상태로 변경될 때 해제되는지 확인 필요 |
| 사용할 수 없는 계정 | **[미확인]** 어떤 상황에 사용 불가 계정이 되는지 확인 필요 |
