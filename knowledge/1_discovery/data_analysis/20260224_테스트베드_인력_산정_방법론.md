## 핵심 발견

오더 데이터 기반으로 테스트베드 인력을 산정할 때의 실전 공식과 전제.

## 산정 공식

```
투입 인원 = (예상 오더 x 커버리지 목표) / (체험 인력 생산성 x 슬롯 시간)
```

## 주요 전제 및 파라미터

| 항목 | 값 | 근거 |
|------|-----|------|
| 슬롯 단위 | 4시간 (10-14 / 14-18 / 18-22) | 체험 인력 집중력 한계 |
| 체험 인력 생산성 | 5~7건/인/4시간 | 숙련 기사(10~12건)의 50~60% |
| 커버리지 목표 | 발생 오더의 60~70% (중간값 65%) | 기존 기사가 나머지 처리 |
| 데이터 기반 | 해당 요일 1개월 평균 | 요일별 편차 반영 |

## 적용 결과

- 4일간 필요 연인원: 약 190명분
- 피크 시간대(18-22시)는 비피크 대비 1.5~1.7배 인력 필요
- 실제 투입 가능 인원(27명) 기준 슬롯 로테이션 설계

## So What - 재사용 가이드

향후 유사 테스트베드 인력 산정 시:
1. `tools/analyze_orders.py`로 해당 지점의 요일/시간대별 오더 분포 추출
2. 위 공식에 파라미터 대입 (생산성은 대상 인력 유형에 따라 조정)
3. 피크/비피크 비율을 반영하여 슬롯별 차등 배치
