  - 첫 기본 상태값은 Off로 한다.
****
**- 신규 오더 제안이 On 일 경우 기사는 추천배차를 통한 제안과 ~~오더 리스트를~~ 통한 직접 배차가 가능하다.**
  - 신규 오더 제안이 Off일 경우에는 추천배차는 해당 기사를 후보 기사로 고려하지 않으며 추가 제안이 없다.
    → 추천 계산 시점에 Off 하여 후보에 포함되더라도, 클라이언트에서 추가로 막거나 추천 시점에 On 여부를 다시 검증한다.
  - 신규 오더 제안이 Off일 경우에는 오더 리스트에 오더를 표시하지 않는다.
    → 리스트 확인을 위해서는 On전환이 필요함을 표시한다.  

**- 기타 사항**  
  - 기사에게 제안되는 오더의 기준: 상점의 관제 지점의 operation type이 NVP인 상점의 오더.
  - 신규 제안 여부 On/Off는 새로운 오더의 배정 가능 여부를 결정할 뿐, 배정받은 오더의 수행에는 영향이 없다.
  - 신규 제안 On 전환 시, 1일 1회에 한하여 안전운행 관련 사항을 팝업으로 안내한다. 
  - On 상태 이후 일정 시간동안 (1)오더 제안에 대한 수락/적극거절 액션, (2)리스트 오더 배정 액션이 없는 경우 자동으로 Off 처리되며 안내 팝업이 발생한다.  |  |  |

    - 추가적으로 묶음오더를 구성하고 있는 오더의 각 픽업지와 원가를 표시한다.
    - 지도와 바텀시트 정보를 파악하여 기사는 수락/거절이 가능하다.

### 주요 화면 설계

- 기사가 오더 수행에 집중할 수 있는 메인화면으로 개편 예정 (수행 중인 오더가 한 건이라도 있는 상태)
- 수행 중인 오더의 순서와 목록, 오더 상세(바텀 시트 형태), 상태 변경, 배차 취소, 신규 오더 제한 여부만 표시
- 목표는 기사가 수행에 최대한 집중하게 하고, 수락률을 높이는 것
- 기사의 현재 위치와 다음 이동 지점만 표시하여 집중도 향상
- 다음 목적지까지 자동 연결되는 직선으로 표시
- 다음 목적지가 상점인 경우 조리완료 잔여/경과 시간 표시, 고객인 경우 어떤 상점 오더를 배송하는지 표시

### 이동 경로 및 주소 표시

- 별도 버튼을 통해 목록 형태로 이동 지점 확인 가능
- 상점의 경우 상점명과 오더 번호, 고객의 경우 동까지만 주소 표시 (블라인드 처리)
- 다음 순서 오더만 상세 주소 확인 가능
- 상점일 경우: 상점명, 상세주소, 상점 전화버튼, 오더 배차 취소 버튼 표시
- 도착지일 경우: 도착지 상세 주소, 오더 상점 정보, 주문 번호, 결제 방식, 결제 금액, 고객 전화 버튼 표시
- 고객 요청사항 추가 필요 (누락되어 있음)

### 액션 및 상태 변경

- 직후 수행해야 할 업무 관련 상태만 변경 가능
- 상점 도착 처리는 일정 범위 내 위치했을 때만 가능 (데이터 정합성 확보 목적)
- 현재 계획은 100m 정도의 반경으로 설정
- GPS 오류 등으로 인한 경우를 위해 바이패스 기능 추가 예정
- 상점 도착 후 음식 픽업 처리는 위치 체크 없이 가능
- 액션 수행 시 로그 남기기로 결정

### 사진 촬영 및 배송 완료

- 선불권 오더 배송완료 시 자동으로 사진촬영 화면으로 이동
- 필요시 화면상 사진촬영 가이드 표시
- 사진 촬영이 필수가 아닌 경우 건너뛸 수 있는 옵션 제공
- 일부 특정 화주사(예: 올리브영) 오더는 사진촬영 필수로 설정
- 수령 중인 화면에 대한 가이드 추가 고려

### 결제 처리

- 후불 결제 오더는 카드기 없이 현금/카드 결제 가능
- 결제 완료 시 자동 배송 완료 처리 여부는 추가 검토 필요
- 현금권의 경우 기사매입 불가 정책으로 운영 예정 (최소한의 현금 거래 지향)

### 배차 취소 정책

- 오더 수행 상태에 따라 배차 취소 가능 여부 달라짐
- 픽업 이전: 기사가 직접 배차 취소 가능
- 픽업 이후: 기사 직접 취소 불가, VCC(관제센터)를 통해서만 가능
- 배차 취소 시 사유 입력 필요 (사전 정의된 프리셋 + 직접 입력 옵션)
- 취소 횟수 제한 관련 논의 (어뷰징 방지 목적)

### 오더 정보 업데이트 및 알림

- 인트라를 통한 오더 상태 변경 시 기사에게 알림 제공
- 오더 추가/취소/상태 변경 시 수행 순서 갱신되고 기사에게 알림
- 바로 다음 행선지는 고정하고 그 이후 순서만 변경 (혼란 방지)
- 도착지 주소가 변경되더라도 수행 순서는 유지

### 신규 오더 제안

- 신규 오더 제안 받을지 여부 온오프 스위치로 설정 가능
- 오프 상태에서 들어온 제안은 표시하지 않고 만료 거절 처리
- 최대 동시 수행 가능 건수는 3개로 제한

### 관리자 권한 제한

- NVP 지정 기사의 오더는 매니저 PC에서 상태 변경 불가
- 매니저 PC에서는 NVP 기사의 위치와 오더 확인만 가능
- NVP 지점 소속 기사들은 매니저 PC 목록에서 표시하지 않을 예정
- NVP 지점 관련 오더는 상태 변경 불가
- 어드민 계정이 매니저PC로 로그인한 경우에도 동일 제약 적용
  - 단, 오더 상태값 기준 `대기(접수 계열)` 오더는 조회 가능
