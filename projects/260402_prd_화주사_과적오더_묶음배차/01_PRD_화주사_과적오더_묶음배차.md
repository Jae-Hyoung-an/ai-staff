## 1. 개요

### 1.1 배경 및 문제 정의

화주사(홈플러스, 스타벅스) 과적 오더는 실제 배송 건이 추가되는 개념이 아니라, 원오더로 배송하는 물건이 일정 크기/용량 등을 초과할 때 기사에게 **+N건만큼의 배송비를 지급하기 위해 생성되는 오더**이다.

**현재 문제:**

- 시스템이 화주사 과적 오더와 원오더를 **별개 오더로 판단**하여, 두 오더를 각각 다른 기사에게 배정한다.
- 원오더를 배정받은 기사는 정상적으로 수행하지만, 과적 오더를 배정받은 기사는 **상점에 도착해도 물건이 없는 상황**을 겪게 된다.
- 이로 인해 **기사 2명이 현장에 도착하는 문제**가 발생하며, 현장 클레임과 화주사 이탈 리스크가 증가한다.

**추가 제약 사항:**

- 화주사 과적 오더의 생성 시점은 원오더 생성 시점과 **얼마나 떨어져 있을지 예측 불가**하다.
- 현재 플러스앱에서는 과적 기능이 **미제공** 상태이며, VCC가 인트라에서 수기로 처리하고 있다 (260319 기준).

### 1.2 목표

- 플러스앱(NVP 기사 전용 앱)에서 화주사 과적 오더를 원오더와 묶어 **한 명의 기사가 수행**하도록 개선한다.
- 과적 오더 생성 시점과 무관하게 **원오더 배정 기사에게 과적 오더가 자동으로 배정**되도록 한다.
- 과적 오더 수행 시 **1회의 상태 변경으로 원오더+과적 오더 전체가 처리**되도록 하여 기사 수행 부담을 줄인다.

### 1.3 적용 대상

| 구분 | 대상 |
|------|------|
| **앱** | 부릉플러스앱 (NVP 직고용 기사 전용) |
| **기사 유형** | NVP 기사 (DIRECTLY_EMPLOYED) |
| **화주사** | 홈플러스, 스타벅스 |
| **오더 유형** | 화주사 과적 오더 (원오더에 종속) |

## 2. 성공 지표

### 핵심 지표 (Core)

- 과적 오더 원오더 기사 불일치율: **0%** 달성 (현재 불일치 발생 중)
- 과적 묶음 오더 배차 성공률: **기존 원오더 배차 성공률과 동일 수준** 유지

### 보조 지표 (Usage)

- 과적 묶음 오더 정상 완료율
- 케이스 1(배정 전 묶음 제안) vs 케이스 2(배정 후 강제 배정) 비율 모니터링

### 가드레일 지표 (Guardrail)

- 기존 일반 오더 배차 성공률: **하락 없음**
- 과적 관련 CS/VCC 문의: **감소** (기존 수기 처리 건 대비)
- 기사 배차 취소율: **증가 없음**

## 3. 범위

### In-Scope (P0)

- AI배차에서 원오더만 배차 시도, 과적 오더 배차 대상 제외
- 배정 전 과적 오더 존재 시 묶음 형태로 기사에게 제안 (합산 원가 표시, 갯수 제한 없음)
- 배정 후 과적 오더 생성 시 동일 기사에게 자동 강제 배정 + 앱 내 안내
- 원오더 상태 변경 시 종속 과적 오더 상태 자동 동기화
- 묶음 해제 조건: **원오더 취소 시에만** 묶음 해제
- 배차 취소 시 묶음 유지 + 다른 플러스앱/프렌즈 기사에게 묶음 재제안
- 접수 상태 전환 시 묶음 유지 + 독점 제안 지속 + 셀프배차 노출 + 2륜 미노출
- 묶음 오더 VCC 배차 대상 제외 (묶음 해제 시 개별 VCC 배차 대상)
- 기존 정책과의 충돌 해소 (동시 수행 건수, 배차 취소 카운트, 셀프배차 등)

### Out-of-Scope

- 이륜 기사앱 대응 (기존 일반오더 취급 유지)
- 플러스앱 내 기사 직접 과적 신청 기능 (기존대로 VCC 수기 처리)
- 화주사 과적 오더 생성 규칙 변경 (기존 홈플러스/스타벅스 규칙 유지)
- 인트라 과적 오더 관리 UI 변경

## 4. 프로덕트 영향범위

| 프로덕트 구분 | 내용 | 비고 |
|--------------|------|------|
| **플러스앱 (신규앱)** | - 기사는 화주사 과적 묶음 오더 제안 시 과적 포함 여부와 합산 원가를 확인할 수 있다<br>- 기사는 원오더 1건의 상태 변경으로 종속 과적 오더를 한번에 처리할 수 있다<br>- 기사는 수행 중 과적 오더 추가 배정 시 안내를 받을 수 있다<br>- 기사는 셀프배차 목록에서 접수 상태의 묶음 오더를 확인하고 배정받을 수 있다 | |
| **배차 엔진 (서버)** | - AI배차는 화주사 과적 오더를 배차 대상에서 제외한다<br>- 원오더 배정 시 기존재 과적 오더를 동일 기사에게 자동 강제 배정한다<br>- 원오더 배정 후 생성되는 과적 오더를 동일 기사에게 자동 강제 배정한다<br>- 배차 취소 시 묶음을 유지한 채 다른 기사에게 재제안한다<br>- 접수 상태에서 묶음을 유지하며 독점 제안을 지속한다<br>- 묶음 오더는 VCC 배차 대상 및 2륜 기사앱 노출에서 제외한다 | |
| **인트라** | - 화주사 과적 묶음 상태의 과적 오더는 개별 상태 변경이 제한된다 (추후 확인 필요)<br>- 묶음 오더는 VCC 배차 관리 목록에서 미노출된다 | |
| **프렌즈앱** | - 배차 취소된 묶음 오더가 프렌즈에게도 제안될 수 있다. 이 경우 프렌즈 기존 과적 묶음 정책을 따른다 | |

## 5. 용어 정의

| 용어 | 정의 |
|------|------|
| **원오더** | 화주사에서 최초 생성된 오더. 과적 오더의 기준이 되는 부모 오더 |
| **화주사 과적 오더** | 화주사별 규칙에 따라 원오더에 종속되어 생성되는 오더. 실제 배송은 원오더 1건이지만, 추가 배송비 지급을 위해 +N건으로 생성됨 |
| **과적 묶음 오더** | 원오더 + 종속된 화주사 과적 오더의 묶음. 원오더가 취소되지 않는 한 묶음 관계를 유지한다 |
| **과적 묶음 해제 오더** | 원오더가 취소되어 묶음이 해제된 과적 오더. 묶음 해제 시 일반 오더로 전환 |
| **NVP 오더** | 오더 생성 상점의 관제 지점이 NVP 지점인 오더 |
| **플러스앱** | NVP 직고용 기사 전용 배달 수행 앱 (클라이언트: `riderplus`) |

## 6. 화주사 과적 오더 생성 규칙 (기존 유지)

### 홈플러스

- 상점에 "홈플러스" 연동 API가 설정되어 있어야 한다.
- 원오더 접수 시간 기준 **24시간 이내**, 원오더의 자체주문번호와 **동일한 값**을 가진 추가 오더는 과적 오더로 생성된다.

### 스타벅스

- 상점에 "스타벅스" 연동 API가 설정되어 있어야 한다.
- 원오더의 자체주문번호 끝자리가 **"1"**로 끝나고, 24시간 이내 자체주문번호 끝자리가 **"2"**이며, 배송 메시지에 **"추가콜"** 단어가 포함된 오더는 과적 오더로 생성된다.

## 7. 기존 정책 충돌 및 정리 사항

### 확정 사항

| # | 기존 정책 | 충돌 내용 | 결정 |
|---|-----------|----------|------|
| 1 | **최대 동시 수행 건수 (3건)** | 과적 오더가 N건 종속될 수 있음, 현행 3건 제한 초과 가능 | **원오더만 카운트**한다. 화주사 과적 오더는 동시 수행 건수에 포함하지 않는다. |
| 2 | **배차 취소 횟수 (일 3회)** | 과적 묶음 오더 배차 취소 시 카운트 기준 | **원오더 기준 1건**으로만 카운트한다. 종속 과적 오더는 별도 카운트하지 않는다. |
| 3 | **셀프배차 목록** | 묶음 오더의 셀프배차 노출 방식 | **플러스앱 셀프배차**: 묶음 상태에서는 원오더만 노출 + [과적] 표시 + 합산 원가 표기. 수락 시 종속 과적 오더도 함께 배차. **프렌즈 셀프배차**: 묶음 오더는 **미노출**. 묶음 해제 시 각각 별개 일반 오더로 양쪽 모두 노출. |
| 4 | **동시제안** | 묶음 오더의 동시제안 여부 | 묶음 오더는 **동시제안을 진행하지 않는다**. 접수 상태에서도 독점 제안만 지속한다. |
| 5 | **무응답 거절 3회 연속 자동 Off** | 과적 묶음 오더 거절 카운트 기준 | 제안은 원오더 기준으로만 나가므로 **1회 거절**로 카운트한다. |
| 6 | **VCC 배차** | 묶음 오더의 VCC 배차 대상 여부 | 묶음 오더가 존재할 경우 원오더+종속 과적 오더 모두 **VCC 배차 대상에서 제외**한다. VCC 배차 관리 목록에서 미노출한다. 묶음이 해제되면 각각 개별 오더로 VCC 배차 대상이 된다. |
| 7 | **접수 상태 전환 시 2륜 기사앱 노출** | 접수 상태로 변경 시 2륜 기사 전투배차 목록 노출 여부 | 묶음 오더는 접수 상태로 변경되더라도 **2륜 기사앱에 노출되지 않는다**. |
