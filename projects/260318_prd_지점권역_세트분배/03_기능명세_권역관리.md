# 03 기능명세 — 권역관리 (P0)

> 인트라 `권역관리` 메뉴의 P0 기능 명세
> 관리정책 생성/상세/수정, 3개 설정 탭, 최종 저장, 버전 관리, 예약 변경 포함

## 2.3 권역관리 — 관리정책 중심 운영 (P0)

### 기본 정책

| 정책 | 설명 |
|------|------|
| **최상단 메뉴** | 인트라 GNB 최상단에 `권역관리` 메뉴 제공 |
| **관리 단위** | 관리정책 단위. 1개 관리정책에 3개 탭 설정값을 묶어 관리 |
| **목록 컬럼** | 관리정책명, 생성일, 마지막 수정일 |
| **목록 검색** | 관리정책명 기준 검색 제공 |
| **생성 진입** | 목록 하단 `관리정책 생성` 버튼 제공 |
| **탭 구성 (생성 시)** | 세트 물량 구성 / 보상기준 / 연결 zone |
| **탭 구성 (상세 시)** | 생성 시 탭 3개 + `버전 관리` + `예약 변경` 탭 추가 노출 |
| **저장 방식** | 각 탭 개별 저장 없음. 어느 탭에 있든 최하단 `관리 정책 저장` 버튼으로 전체 저장 |
| **필수 입력 규칙** | 모든 탭 항목 빈칸 허용 안 함 (0은 허용) |
| **시간 정합성 규칙** | 시작 < 종료 + 시간대 순서 정렬 + 오버랩 금지 + 입력된 구간 내 5개 시간대 연속 |
| **정책명 중복 규칙** | 기존 관리정책명과 중복 불가 (저장 시 검증) |
| **오류 안내 방식** | 저장 실패 시 첫 번째 오류 1건만 안내 |
| **Zone 연결 규칙** | 다른 관리정책에 이미 연결된 zone은 연결 불가 |
| **버전 생성 시점** | 관리 정책 저장 시 전체 설정(3탭)을 새 버전으로 자동 스냅샷 |
| **버전 상태** | 현재 적용 / 예약 / 이전 버전 / 취소됨 |
| **버전 보관** | 기간/수량 제한 없이 전체 보관 |
| **예약 변경 단위** | 주간 (적용 시작 일시: 매주 월요일 00:00 기본, 날짜 및 시간 변경 가능) |
| **예약 제한** | 동시에 1건의 예약만 허용 (새 예약 시 기존 예약 덮어쓰기) |
| **예약 적용 실패 알림** | 적용 실패 시 슬랙을 통해 실무자에게 알림 발송 |

### 사용자 시나리오

> **시나리오 1: 목록 조회 및 검색**
> 1. RM이 인트라 > 권역관리 메뉴에 진입한다.
> 2. 목록에서 관리정책명, 생성일, 마지막 수정일을 확인한다.
> 3. 상단 검색창에 관리정책명을 입력해 목록을 필터링한다.

> **시나리오 2: 관리정책 생성 및 전체 저장**
> 1. RM이 목록 하단 `관리정책 생성` 버튼을 누른다.
> 2. 생성 페이지에서 관리정책명(필수)을 입력한다.
> 3. 3개 탭의 값을 모두 입력한다.
> 4. 어느 탭에서든 하단 `관리 정책 저장` 버튼을 누른다.
> 5. 시스템이 3개 탭 전체를 검증한다.
> 6. 성공 시 성공 안내 후 목록으로 복귀하고, 방금 생성한 정책이 목록에 표시된다.
> 7. 실패 시 첫 번째 오류 사유를 안내한다.

> **시나리오 3: 상세 조회 및 수정**
> 1. RM이 목록에서 특정 관리정책을 선택한다.
> 2. 상세 페이지에서 현재 저장된 3개 탭 값을 확인한다.
> 3. 필요한 값을 수정 후 `관리 정책 저장` 버튼으로 전체 저장한다.
> 4. 저장 완료 시 변경 사항이 반영되고, 새 버전이 자동 생성된다.

> **시나리오 4: 이전 버전 조회 및 비교**
> 1. RM이 생성 완료된 관리정책 상세 페이지에 진입한다.
> 2. `버전 관리` 탭을 선택한다.
> 3. 버전 목록에서 과거 버전을 선택해 당시 전체 설정값을 읽기 전용으로 확인한다.
> 4. `현재 버전과 비교` 버튼을 눌러 현재 적용 중인 버전과 선택 버전의 차이를 확인한다.

> **시나리오 5: 주간 예약 변경 생성**
> 1. RM이 상세 페이지에서 `예약 변경` 탭을 선택한다.
> 2. 현재 적용 중인 설정값이 복사되어 표시된다.
> 3. 변경이 필요한 설정값을 수정한다.
> 4. 적용 시작 일시(날짜 + 시간)를 지정한다.
> 5. `예약 저장` 버튼을 눌러 예약을 등록한다.
> 6. 버전 관리 탭에서 해당 버전이 "예약" 상태로 표시된다.

> **시나리오 6: 예약 변경 취소**
> 1. RM이 `버전 관리` 탭에서 "예약" 상태의 버전을 선택한다.
> 2. `예약 취소` 버튼을 눌러 예약을 취소한다.
> 3. 해당 버전의 상태가 "취소됨"으로 변경된다.
