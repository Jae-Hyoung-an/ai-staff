# 03 기능명세 — 세트분배 관리 (P0)

> 인트라 `세트분배 관리` 메뉴의 P0 기능 명세
> Zone별 벤더 연결 및 벤더별 세트 물량 분배 관리, 버전 관리, 예약 변경 포함

## 2.7 세트분배 관리 — Zone별 벤더 연결 + 세트 분배 (P0)

### 기본 정책

| 정책 | 설명 |
|------|------|
| **GNB 위치** | 인트라 GNB에 `세트분배 관리` 메뉴 제공 |
| **관리 단위** | Zone 단위. Zone별로 연결 벤더 목록과 벤더별 세트 수를 관리 |
| **목록 컬럼** | Zone 이름, 연결 벤더 수(N개), 수행 세트 수(N개) |
| **목록 검색** | Zone 이름 기준 검색 |
| **상세 탭 구성** | 연결 벤더 / 벤더별 세트 분배 / 버전 관리 / 예약 변경 (진입 시 `연결 벤더` 탭 기본 표시) |
| **벤더 중복 연결 불가** | 다른 Zone에 이미 연결된 벤더는 연결 불가. 저장 시 시스템 검증 |
| **벤더 검색 범위** | 전체 벤더를 검색 대상으로 사용 |
| **세트 수 입력 규칙** | 저장된 값 없는 벤더는 0이 기본. 음수/빈칸 불가. 0 이상 정수 허용 |
| **세트 수 상한** | 상한 없음 |
| **버전 생성 시점** | 연결 벤더 또는 세트 분배 저장 시 전체 설정(연결 벤더 + 세트 분배)을 새 버전으로 자동 스냅샷 |
| **버전 상태** | 현재 적용 / 예약 / 이전 버전 / 취소됨 |
| **버전 보관** | 기간/수량 제한 없이 전체 보관 |
| **예약 변경 단위** | 주간 (적용 시작 일시: 매주 월요일 00:00 기본, 날짜 및 시간 변경 가능) |
| **예약 제한** | 동시에 1건의 예약만 허용 (새 예약 시 기존 예약 덮어쓰기) |
| **예약 적용 실패 알림** | 적용 실패 시 슬랙을 통해 실무자에게 알림 발송 |
| **벤더 상태 종류** | 대기 / 운영중 / 운영중지 / 해지 |
| **벤더 상태 표시** | 연결 벤더 / 벤더별 세트 분배 / 버전 관리 / 예약 변경 — 4개 탭 모두 벤더명 옆에 상태 배지 표시 |
| **비운영 벤더 세트 제한** | 운영중이 아닌 벤더는 세트 수 0만 허용 (벤더별 세트 분배 탭, 예약 변경 탭 공통) |
| **예약 적용 시 상태 보정** | 예약 적용 시점에 운영중이 아닌 벤더의 분배 세트를 자동으로 0개로 변경 후 적용 |

### 사용자 시나리오

> **시나리오 1: Zone 목록 조회 및 검색**
> 1. RM이 인트라 > 세트분배 관리 메뉴에 진입한다.
> 2. Zone 목록(Zone 이름, 연결 벤더 수, 수행 세트 수)이 표시된다.
> 3. 상단 검색란에 Zone 이름을 입력해 목록을 필터링한다.

> **시나리오 2: 연결 벤더 관리**
> 1. RM이 목록에서 특정 Zone을 선택해 상세 페이지로 진입한다.
> 2. `연결 벤더` 탭(기본)에서 현재 연결된 벤더 목록을 확인한다.
> 3. 특정 벤더의 삭제 버튼을 눌러 목록에서 제거한다.
> 4. 하단 `벤더 검색` 버튼으로 새 벤더를 검색해 목록에 추가한다.
> 5. `연결 벤더 목록 저장` 버튼을 누른다.
> 6. 성공 시 성공 팝업 표시. 저장 시 새 버전이 자동 생성된다.
> 7. 실패 시(다른 Zone에 이미 연결된 벤더 포함) 어떤 벤더가 어떤 Zone에 연결되어 있는지 팝업에 표시.

> **시나리오 3: 벤더별 세트 분배**
> 1. RM이 `벤더별 세트 분배` 탭을 선택한다.
> 2. 연결 벤더 탭에서 저장한 벤더 목록이 표시된다.
> 3. 각 벤더 우측에 세트 수를 입력한다 (기본값 0).
> 4. `세트 분배 저장` 버튼을 누른다.
> 5. 음수/빈칸이 없으면 저장 성공. 저장 시 새 버전이 자동 생성된다.

> **시나리오 4: 이전 버전 조회 및 비교**
> 1. RM이 Zone 상세 페이지에서 `버전 관리` 탭을 선택한다.
> 2. 버전 목록에서 과거 버전을 선택해 당시 연결 벤더 + 세트 분배 설정값을 읽기 전용으로 확인한다.
> 3. `현재 버전과 비교` 버튼을 눌러 현재 적용 중인 버전과 선택 버전의 차이를 확인한다.

> **시나리오 5: 주간 예약 변경 생성**
> 1. RM이 Zone 상세 페이지에서 `예약 변경` 탭을 선택한다.
> 2. 현재 적용 중인 연결 벤더 + 세트 분배 설정값이 복사되어 표시된다.
> 3. 변경이 필요한 설정값(벤더 추가/삭제, 세트 수 변경)을 수정한다.
> 4. 적용 시작 일시(날짜 + 시간)를 지정한다.
> 5. `예약 저장` 버튼을 눌러 예약을 등록한다.
> 6. 버전 관리 탭에서 해당 버전이 "예약" 상태로 표시된다.

> **시나리오 6: 예약 변경 취소**
> 1. RM이 `버전 관리` 탭에서 "예약" 상태의 버전을 선택한다.
> 2. `예약 취소` 버튼을 눌러 예약을 취소한다.
> 3. 해당 버전의 상태가 "취소됨"으로 변경된다.

> **시나리오 7: 비운영 벤더 세트 입력 시도**
> 1. RM이 `벤더별 세트 분배` 탭 또는 `예약 변경` 탭에서 벤더별 세트 수를 수정한다.
> 2. 운영중이 아닌 벤더(대기/운영중지/해지)의 세트 수를 1 이상으로 변경하려고 시도한다.
> 3. 시스템이 해당 벤더는 운영중 상태가 아니므로 세트 수를 0만 설정할 수 있다는 안내 메시지를 표시한다.
> 4. 해당 벤더의 세트 수는 0으로 유지된다.

## So What / Next Step

- 연결 벤더와 세트 분배를 탭으로 분리하면, 벤더 지정과 물량 배분 의사결정을 단계적으로 처리할 수 있습니다.
- 벤더 중복 연결 검증이 저장 시점에 발생하므로, 실패 팝업에서 충돌 정보를 명확히 안내하는 것이 중요합니다.
- 버전 관리 도입으로 Zone별 벤더/세트 분배 변경 이력을 전체 스냅샷으로 추적할 수 있어, 설정 변경 영향을 사전/사후에 파악할 수 있습니다.
- 주간 예약 변경을 통해 전주 실적 기반의 선제적 세트 분배 조정이 가능해지며, 매주 월요일 수동 변경 부담을 줄일 수 있습니다.
- 예약 적용 실패 시 슬랙 알림으로 즉시 인지 가능하므로, 세트 분배 공백을 최소화할 수 있습니다.
- 벤더 상태(대기/운영중/운영중지/해지)를 4개 탭 전체에 표시하여 RM이 현재 상태를 즉시 파악할 수 있으며, 비운영 벤더에 대한 세트 분배 제한으로 운영 오류를 사전 방지할 수 있습니다.
- 예약 적용 시 벤더 상태 자동 보정 로직을 통해, 예약 생성 후 벤더 상태가 변경되더라도 비운영 벤더에게 물량이 분배되는 상황을 방지합니다.
