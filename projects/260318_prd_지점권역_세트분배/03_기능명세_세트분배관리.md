## 2.7 세트분배 관리 — Zone별 벤더 연결 + 세트 분배 (P0)

### 기본 정책

| 정책 | 설명 |
|------|------|
| **GNB 위치** | 인트라 GNB에 `세트분배 관리` 메뉴 제공 |
| **관리 단위** | Zone 단위. Zone별로 연결 벤더 목록과 벤더별 세트 수를 관리 |
| **목록 컬럼** | Zone 이름, 연결 벤더 수(N개), 수행 세트 수(N개) |
| **목록 검색** | Zone 이름 기준 검색 |
| **상세 탭 구성** | 연결 벤더 / 벤더별 세트 분배 (진입 시 `연결 벤더` 탭 기본 표시) |
| **벤더 중복 연결 불가** | 다른 Zone에 이미 연결된 벤더는 연결 불가. 저장 시 시스템 검증 |
| **벤더 검색 범위** | 전체 벤더를 검색 대상으로 사용 |
| **세트 수 입력 규칙** | 저장된 값 없는 벤더는 0이 기본. 음수/빈칸 불가. 0 이상 정수 허용 |
| **세트 수 상한** | 상한 없음 |

---

### 사용자 시나리오

> **시나리오 1: Zone 목록 조회 및 검색**
> 1. RM이 인트라 > 세트분배 관리 메뉴에 진입한다.
> 2. Zone 목록(Zone 이름, 연결 벤더 수, 수행 세트 수)이 표시된다.
> 3. 상단 검색란에 Zone 이름을 입력해 목록을 필터링한다.

> **시나리오 2: 연결 벤더 관리**
> 1. RM이 목록에서 특정 Zone을 선택해 상세 페이지로 진입한다.
> 2. `연결 벤더` 탭(기본)에서 현재 연결된 벤더 목록을 확인한다.
> 3. 특정 벤더의 삭제 버튼을 눌러 목록에서 제거한다.
> 4. 하단 `벤더 검색` 버튼으로 새 벤더를 검색해 목록에 추가한다.
> 5. `연결 벤더 목록 저장` 버튼을 누른다.
> 6. 성공 시 성공 팝업 표시.
> 7. 실패 시(다른 Zone에 이미 연결된 벤더 포함) 어떤 벤더가 어떤 Zone에 연결되어 있는지 팝업에 표시.

> **시나리오 3: 벤더별 세트 분배**
> 1. RM이 `벤더별 세트 분배` 탭을 선택한다.
> 2. 연결 벤더 탭에서 저장한 벤더 목록이 표시된다.
> 3. 각 벤더 우측에 세트 수를 입력한다 (기본값 0).
> 4. `세트 분배 저장` 버튼을 누른다.
> 5. 음수/빈칸이 없으면 저장 성공.
