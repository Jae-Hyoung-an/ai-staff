# 기본 규칙

> 이 파일은 항상 적용되는 기본 규칙입니다.
> AGENTS.md에서 참조합니다.

---

## 폴더 구조 가이드

### knowledge/ - 지식 베이스

**용도**: 문서, 메모, 참고자료, 학습 내용

**파일 예시**:
- `my_notes.md` - 개인 메모
- `2026-01-02_meeting_notes.md` - 날짜가 있는 문서
- `project_overview.md` - 프로젝트 설명

**성장 시 구조**:
```
knowledge/
├── notes/          <- 메모, 아이디어
├── references/     <- 참고자료, 외부 정보
├── projects/       <- 프로젝트별 문서
└── learnings/      <- 학습 내용
```

---

### tools/ - 도구 및 자동화

**용도**: 반복 작업을 자동화하는 스크립트

**규칙**:
- 각 도구는 **사용법을 한글로** 상단에 설명
- 복잡한 도구는 별도 `README.md` 작성
- 사용자가 이해할 수 있는 수준으로 설명

**파일 예시**:
```
tools/
├── README.md              <- 도구 목록 및 사용법
├── organize_files.py      <- 파일 정리 스크립트
└── generate_report.py     <- 보고서 생성 스크립트
```

**도구 파일 상단 예시**:
```python
"""
파일 정리 도구

사용법:
  python organize_files.py

하는 일:
  - knowledge/ 폴더의 파일들을 날짜별로 정리
  - 오래된 파일은 archive/ 폴더로 이동
"""
```

---

### rules/ - 확장 규칙

**용도**: 영역별로 필요한 추가 규칙

**성장 시 추가될 수 있는 파일**:
- `writing.md` - 문서 작성 스타일 가이드
- `tools.md` - 도구 개발 규칙
- `{project}.md` - 특정 프로젝트 규칙

## 파일 명명 규칙

| 유형 | 형식 | 예시 |
|------|------|------|
| 일반 문서 | `제목.md` | `project_plan.md` |
| 날짜 문서 | `YYYYMMDD_제목.md` | `20260102_meeting.md` |
| 순서 문서 | `##_제목.md` | `01_introduction.md` |

---

## 문서 작성 기본

### 문서 상단 템플릿

```markdown
# 제목

> 한 줄 설명

---

(본문)
```

### 마크다운 스타일

- 제목: `#`, `##`, `###` 계층 준수
- 목록: `-` 사용 (숫자 목록은 순서가 중요할 때만)
- 코드: 백틱 3개로 감싸기, 언어 명시
- 표: 비교나 정리에 적극 활용

## 버전 관리 (선택)

Git을 사용한다면:
- 커밋 메시지는 한글로, 무엇을 왜 했는지 명시
- 예: `knowledge/ 폴더 구조 정리 - 날짜별 하위 폴더 생성`

Git을 사용하지 않는다면:
- 중요한 변경 전 파일 백업 (복사본 생성)

---

## 이 규칙의 확장

이 파일은 기본 규칙입니다. 사용하면서 필요한 규칙이 생기면:

1. 특정 영역에 해당하면 → `rules/{영역}.md` 새로 생성
2. 전체 적용이면 → 이 파일에 추가
3. AI와 대화하며 함께 결정
