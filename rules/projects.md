# 프로젝트 관리 규칙

> AI와 함께 만들어낸 산출물(리포트, 웹사이트, 발표자료, 글 등)을 관리하는 규칙

---

## 폴더 구조

```
projects/
├── _scratch/              <- 임시/adhoc 산출물 (간단한 것들)
│   ├── 260107_간단요약.md
│   └── 260108_회의정리.md
├── YYMMDD_프로젝트명/     <- 체계적 프로젝트
│   ├── README.md          <- 프로젝트 설명 (선택)
│   ├── 주요_산출물.md
│   └── assets/            <- 이미지, 첨부파일 등
└── ...
```

### _scratch/ (임시 공간)

**용도:** 간단한 정리, 메모, adhoc 산출물

| 특징 | 설명 |
|------|------|
| 폴더 없이 단일 파일 | `_scratch/260107_간단요약.md` |
| 정리 부담 없음 | 쌓여도 OK |
| 나중에 승격 가능 | 중요해지면 → 정식 프로젝트 폴더로 이동 |

**파일 네이밍:** `YYMMDD_제목.md`

---

## 네이밍 컨벤션

### 폴더명

```
YYMMDD_프로젝트명/
```

| 요소 | 규칙 | 예시 |
|------|------|------|
| **YYMMDD** | 프로젝트 시작일 (6자리) | `260107` |
| **프로젝트명** | 한글 또는 영어, 언더스코어 구분 | `시장조사_리포트` |

**예시:**
- `260107_시장조사_리포트/`
- `260115_회사소개_웹사이트/`
- `260120_2월_발표자료/`
- `260125_블로그_ai_협업/`

### 파일명

- 프로젝트 내 파일: `설명적_이름.확장자`
- 버전 관리 필요 시: `파일명_v1.md`, `파일명_v2.md`

## 프로젝트 유형별 구조

### 리포트/문서

```
260107_시장조사_리포트/
├── report.md              <- 메인 리포트
├── summary.md             <- 요약본 (선택)
└── assets/
    └── charts/            <- 차트, 이미지
```

### 웹사이트

```
260115_회사소개_웹사이트/
├── index.html
├── styles.css
├── script.js
└── assets/
    ├── images/
    └── fonts/
```

### 발표자료

```
260120_2월_발표자료/
├── slides.html            <- 슬라이드 (또는 .md)
├── speaker_notes.md       <- 발표 노트
└── assets/
    └── images/
```

### 글/콘텐츠

```
260125_블로그_ai_협업/
├── post.md                <- 본문
├── draft.md               <- 초안 (선택)
└── assets/
    └── cover.png
```

## AI 행동 규칙

### projects/ 사용 트리거

AI는 아래 조건에서 **자동으로 projects/ 생성을 제안**합니다:

| 트리거 | 예시 | AI 행동 |
|--------|------|---------|
| **복합 산출물** | "웹사이트 만들어줘", "발표자료 만들어줘" | 바로 `projects/YYMMDD_제목/` 생성 |
| **명시적 결과물** | "리포트 작성해줘", "분석 보고서 만들어줘" | "projects/에 저장할까요?" 제안 |
| **공유 의도 언급** | "고객에게 보낼 자료", "발표할 자료" | `projects/YYMMDD_제목/` 생성 |
| **반복 작업 예상** | "시리즈로 글 써줘", "이어서 작업할 거야" | `projects/YYMMDD_제목/` 생성 |
| **보관 의도** | "나중에 다시 볼", "저장해둬" | projects/ 생성 제안 |
| **간단한 정리** | "빠르게 정리해줘", "메모해둬", "간단히 문서화" | `projects/_scratch/`에 저장 |

**projects/ 불필요한 경우:**

| 상황 | AI 행동 |
|------|---------|
| 단순 질문 ("~가 뭐야?") | 채팅에서 바로 응답 |
| 일회성 수정 ("이 문장 고쳐줘") | 채팅에서 바로 응답 |
| 요약 요청 ("이거 요약해줘") | 채팅에서 바로 응답 |
| knowledge/ 문서 작업 | knowledge/에 저장 |

**제안 형식:**

```
📁 이 작업은 여러 파일이 생성될 것 같아요.
→ projects/260107_프로젝트명/ 폴더를 만들어서 작업할까요?

[예] [아니오, 여기서 진행]
```

### 프로젝트 생성 시

| 상황 | AI 행동 |
|------|---------|
| "리포트 만들어줘" | `projects/YYMMDD_제목/` 폴더 생성 후 작업 |
| "웹사이트 만들어줘" | `projects/YYMMDD_제목/` + 기본 구조 생성 |
| "발표자료 만들어줘" | `projects/YYMMDD_제목/` + slides 파일 생성 |
| "블로그 글 써줘" | `projects/YYMMDD_제목/post.md` 생성 |

### 기존 프로젝트 작업 시

| 상황 | AI 행동 |
|------|---------|
| "이어서 작업하자" | 가장 최근 프로젝트 확인 후 작업 |
| "260107 프로젝트 열어줘" | 해당 날짜 prefix 프로젝트 찾아서 작업 |
| 프로젝트 여러 개일 때 | 목록 보여주고 선택 요청 |

### README.md 권장 (선택)

복잡한 프로젝트는 README.md 추가:

```markdown
# 프로젝트 제목

## 목적
이 프로젝트가 해결하려는 것

## 산출물
- `report.md` - 메인 리포트
- `summary.md` - 요약본

## 상태
- [ ] 초안 작성
- [x] 1차 리뷰 완료
- [ ] 최종본

## 메모
작업 중 참고사항
```

## knowledge/ vs projects/ 구분

| 폴더 | 역할 | 예시 |
|------|------|------|
| **knowledge/** | 참조할 지식 (Input) | 시장 동향 자료, 경쟁사 분석 |
| **projects/** | 만들어낸 결과물 (Output) | 내가 쓴 리포트, 만든 웹사이트 |

> 💡 projects/에서 만든 것 중 나중에 참조가치가 있으면 knowledge/로 이동

## 정리 규칙

### 완료된 프로젝트

- 삭제하지 않고 그대로 보관
- 필요시 `_archive/` 하위 폴더로 이동

### 오래된 프로젝트 정리

| 상황 | AI 제안 |
|------|---------|
| 3개월 이상 미사용 | "archive로 옮길까요?" |
| 임시/실험 프로젝트 | 완료 후 삭제 제안 |
