# 온보딩 프로세스

> 이 규칙은 `knowledge/` 폴더가 비어있거나 파일이 3개 미만일 때 자동으로 실행됩니다.

---

## 트리거 조건

```
knowledge/ 폴더가:
- 존재하지 않음, 또는
- 비어있음, 또는
- 파일이 3개 미만
```

→ 온보딩 시작

---

## 온보딩 흐름

```
┌─────────────────────────────────────────────────────────┐
│  1. 환영 인사 & 온보딩 안내                              │
│     ↓                                                   │
│  2. 기본 정보 수집 (역할, 주요 업무)                      │
│     ↓                                                   │
│  3. 목표 수집 (AI와 함께 이루고 싶은 것 Top 3~5) ⭐       │
│     ↓                                                   │
│  4. 스타일 수집 (응답 선호, 용어, 피할 표현)              │
│     ↓                                                   │
│  5. 조직 컨텍스트 (팀 구조, 협업, 의사결정)               │
│     ↓                                                   │
│  6. 도메인 지식 (핵심 개념, 참고 자료)                    │
│     ↓                                                   │
│  7. rules/user.md 생성                                  │
│     ↓                                                   │
│  8. knowledge/ 폴더 구조 제안                            │
│     ↓                                                   │
│  9. 온보딩 완료 & 첫 작업 시작                           │
└─────────────────────────────────────────────────────────┘
```

---

## 단계별 상세

### Step 1: 환영 인사 & 안내

**AI가 말할 내용:**

```
안녕하세요! 👋

이 공간에서 AI와 함께 효과적으로 일하기 위해,
먼저 몇 가지를 여쭤볼게요. (약 5분)

수집된 정보는 `rules/user.md`에 저장되어,
앞으로 대화할 때 참고됩니다.

시작할까요?
```

---

### Step 2: 기본 정보 수집

**질문:**

```
1. 어떤 역할/직책을 맡고 계신가요?
   (예: HR 담당자, 전략 기획자, PM, 디자이너...)

2. 주요 업무는 무엇인가요?
   (예: 채용 프로세스 관리, 시장 분석, 제품 기획...)
```

**저장 위치:** `rules/user.md` > 사용자 프로필

---

### Step 3: 목표 수집 ⭐ (핵심)

**질문:**

```
3. 이 공간에서 AI와 함께 이루고 싶은 것을 3~5개 말씀해주세요.
   
   구체적일수록 좋아요. 예시:
   - "채용 JD를 더 빠르게 작성하고 싶다"
   - "시장 분석 보고서를 체계적으로 관리하고 싶다"
   - "회의록 작성 시간을 줄이고 싶다"
   - "디자인 스펙 문서를 일관성 있게 관리하고 싶다"
   
   이 목표들이 앞으로 AI의 지향점이 됩니다.
```

**저장 위치:** `rules/user.md` > AI와 함께 이루고 싶은 것

**중요성:**
- AI가 관련 없는 방향으로 가면 이 목표로 되돌아옴
- 대화 중 우선순위 판단 기준
- 진척 상황 추적 가능

---

### Step 4: 스타일 수집

**질문:**

```
4. AI가 어떤 스타일로 응답하면 좋을까요?

   a) 분량: 간결하게 / 상세하게 / 상황에 따라
   b) 형식: 글 위주 / 표 위주 / 목록 위주 / 혼합
   c) 톤: 전문적 / 친근 / 중립
   
5. 자주 사용하는 용어가 있다면 알려주세요.
   (예: "우리 팀에서 KB는 Knowledge Base를 의미해요")

6. 피했으면 하는 표현이 있나요?
   (예: "이모지 쓰지 마", "~할 것임 대신 ~합니다로")
```

**저장 위치:** `rules/user.md` > 커뮤니케이션 선호

---

### Step 5: 조직 컨텍스트 (선택)

**질문:**

```
7. (선택) 소속 조직/팀에 대해 간단히 알려주세요.
   - 팀 이름, 규모
   - 주요 협업 대상
   - 보고/승인 라인

   → 이 정보가 있으면 문서 작성 시 맥락을 더 잘 반영할 수 있어요.
   → 건너뛰어도 됩니다.
```

**저장 위치:** `rules/user.md` > 도메인 컨텍스트 > 조직 구조

---

### Step 6: 도메인 지식 (선택)

**질문:**

```
8. (선택) 업무에서 중요한 핵심 개념이나 용어가 있나요?
   (예: "우리 회사에서 '전환율'은 유료 구독 전환을 의미해요")

9. (선택) 참고할 만한 기존 문서가 있나요?
   → 있다면 나중에 knowledge/에 추가하면 됩니다.

10. (선택) 반복적으로 하는 작업 패턴이 있나요?
    (예: "매주 월요일 주간 보고서 작성", "분기별 시장 분석")
```

**저장 위치:** `rules/user.md` > 도메인 컨텍스트

---

### Step 7: rules/user.md 생성

**AI 행동:**

1. 수집된 정보를 `rules/user.md` 형식에 맞게 정리
2. 사용자에게 확인 요청
3. 승인 후 파일 생성

```
수집된 정보를 정리했어요. 확인해주세요:

[정리된 내용 미리보기]

이대로 저장할까요? 수정할 부분이 있으면 말씀해주세요.
```

---

### Step 8: knowledge/ 폴더 구조 제안

**AI 행동:**

수집된 역할과 목표를 기반으로 맞춤형 폴더 구조 제안.

**예시 (HR 담당자):**

```
말씀하신 역할과 목표를 바탕으로, 이런 폴더 구조를 제안드려요:

knowledge/
├── recruiting/       <- 채용 관련 (JD, 면접 질문 등)
├── policies/         <- HR 정책 문서
├── team/             <- 팀/조직 정보
└── references/       <- 참고 자료

이 구조로 시작할까요? 다르게 하고 싶은 부분이 있으면 말씀해주세요.
```

**예시 (전략 기획자):**

```
knowledge/
├── market/           <- 시장 분석
├── competitors/      <- 경쟁사 분석
├── strategy/         <- 전략 문서
└── references/       <- 참고 자료
```

---

### Step 9: 온보딩 완료 & 첫 작업

**AI가 말할 내용:**

```
온보딩이 완료되었어요! 🎉

저장된 파일:
- rules/user.md (당신의 프로필과 선호도)

생성된 폴더:
- knowledge/[구조]

이제 첫 번째 목표부터 시작해볼까요?
[목표 #1]: [사용자가 말한 첫 번째 목표]

어떻게 도와드릴까요?
```

## 온보딩 규칙

### 필수 vs 선택

| 단계 | 필수 여부 | 건너뛰기 가능 |
|------|----------|--------------|
| Step 2 (기본) | ✅ 필수 | ❌ |
| Step 3 (목표) | ✅ 필수 | ❌ |
| Step 4 (스타일) | ✅ 필수 | ❌ |
| Step 5 (조직) | ⚪ 선택 | ✅ |
| Step 6 (도메인) | ⚪ 선택 | ✅ |

### 사용자가 건너뛰기 원할 때

```
"나중에 할게" / "건너뛰기" / "skip" 등 감지 시:

→ 필수 항목: "이 부분은 AI가 잘 도와드리기 위해 필요해요. 간단히만 답해주셔도 됩니다."
→ 선택 항목: "알겠습니다. 나중에 필요하면 추가할 수 있어요."
```

### 정보가 부족할 때

최소 정보만 있어도 시작 가능:
- 역할 (한 줄)
- 주요 업무 (한 줄)
- 목표 1개

나머지는 사용하면서 점진적으로 수집.

---

## 온보딩 후 진화

온보딩은 시작일 뿐. 사용하면서 다음이 자동으로 진화:

| 항목 | 진화 방식 |
|------|----------|
| `rules/user.md` | 반복 패턴 발견 시 "추가할까요?" 알림 |
| `knowledge/` | 파일이 쌓이면 하위 폴더 분리 제안 |
| `rules/` | 같은 패턴 3회 반복 시 새 규칙 제안 |
